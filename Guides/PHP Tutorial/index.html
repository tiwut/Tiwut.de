A Comprehensive Beginner's Guide to PHP

Introduction to PHP

PHP (Hypertext Preprocessor) is a widely-used open source general-purpose scripting language that is especially suited for web development and can be embedded into HTML. PHP code is executed on the server, and the result (often HTML) is sent back to the client's browser.

Key Features of PHP:

  • Server-Side Scripting: PHP scripts are processed on the web server before the page is sent to the user.
  • Widely Used: Powers a significant portion of the web, including popular Content Management Systems (CMS) like WordPress, Drupal, and Joomla.
  • Database Integration: Excellent support for various databases like MySQL, PostgreSQL, SQLite, etc.
  • Cross-Platform: Can run on various operating systems (Windows, Linux, macOS).
  • Easy to Learn: PHP has a relatively gentle learning curve for beginners familiar with C-like syntax.
  • Large Community & Documentation: Extensive resources and a large community provide ample support.

PHP is used for creating dynamic web pages, handling forms, working with databases, managing sessions and cookies, file manipulation, and much more.

Setting Up Your Environment

To run PHP scripts, you need a web server with PHP installed and usually a database server like MySQL.

1. Local Development Environment (Recommended)

The easiest way to get started is by installing a software stack that includes Apache (web server), PHP, and MySQL. Popular choices:

  • XAMPP: Cross-platform (Windows, macOS, Linux). Download XAMPP
  • MAMP: Primarily for macOS, but also available for Windows. Download MAMP
  • WAMP (WampServer): For Windows. Download WampServer
  • Laragon (Windows): A modern, fast, and easy-to-use universal development environment. Download Laragon

After installing one of these, you'll typically place your PHP files in a specific web root directory (e.g., htdocs for XAMPP, www for WampServer/Laragon) and access them via your browser using a local URL like http://localhost/your-file.php.

2. Text Editor or IDE

You'll need a code editor to write your PHP scripts.

  • Visual Studio Code (VS Code): Excellent PHP support with extensions like "PHP Intelephense" or "PHP IntelliSense".
  • PhpStorm: A powerful, paid IDE specifically for PHP development by JetBrains (free for students/educators).
  • Sublime Text, Atom, Notepad++: Good general-purpose code editors with PHP syntax highlighting.

3. Running PHP from Command Line (CLI)

If PHP is installed and in your system's PATH, you can also run PHP scripts directly from the command line (terminal):

php your_script.php

To check your PHP installation and version from the command line:

php -v

Basic PHP Syntax

PHP scripts are typically embedded within HTML documents, but they can also exist as standalone files.

PHP Tags

PHP code is enclosed in special start and end tags that allow you to jump into and out of "PHP mode."

  • Standard tags: <?php ... ?> (Recommended)
  • Short echo tag: <?= ... ?> (Equivalent to <?php echo ... ?>. Usually enabled.)
  • Short tags: <? ... ?> (Discouraged, may not be enabled by default.)
<!DOCTYPE html>
<html>
<head>
    <title>My PHP Page</title>
</head>
<body>
    <h1>Welcome!</h1>
    <?php
        echo "<p>This text is generated by PHP.</p>";
    ?>

    <p>Today's date is: <?= date('Y-m-d') ?></p>
</body>
</html>

Statements and Semicolons

Like C or Java, PHP statements are terminated with a semicolon (;).

<?php
    $message = "Hello"; // A statement
    echo $message;     // Another statement
?>

The closing PHP tag ?> automatically implies a semicolon for the last statement, but it's good practice to include it.

Comments

  • Single-line comments: // ... or # ...
  • Multi-line comments: /* ... */
<?php
    // This is a single-line comment
    $name = "Alice"; # This is also a single-line comment

    /*
    This is a
    multi-line comment block.
    */
    echo $name;
?>

Case Sensitivity

  • Variable names are case-sensitive ($name and $Name are different).
  • Function names, class names, and keywords (e.g., echo, if, while) are generally NOT case-sensitive. However, it's good practice to stick to a consistent casing (usually lowercase for keywords, camelCase or PascalCase for functions/classes).

Variables and Data Types

Variables

Variables in PHP start with a dollar sign ($) followed by the variable name.

  • A variable name must start with a letter or an underscore (_).
  • It can only contain alpha-numeric characters and underscores (A-z, 0-9, and _).
  • It is case-sensitive.
  • PHP is a loosely typed language, meaning you don't need to declare the data type of a variable in advance. The type is determined at runtime based on the value assigned.
<?php
    $text = "Hello PHP!";
    $number = 123;
    $float_number = 10.5;
    $_myVariable = true;

    echo $text;
    echo "<br>";
    echo $number;
?>

Data Types

PHP supports several data types:
  • String: Sequence of characters. Enclosed in single ('') or double ("") quotes. Double quotes allow variable interpolation and escape sequences (e.g., \n). 
    $name = "Alice";
$greeting = "Hello, $name!"; // $name is interpolated
$message = 'Hello, $name!'; // $name is treated literally
  • Integer: Whole numbers (e.g., -5, 0, 100).
  • Float (Floating-point numbers, also called double): Numbers with a decimal point (e.g., 3.14, -0.5).
  • Boolean: Can be either true or false (case-insensitive keywords). 
    $is_active = true;
$is_admin = FALSE;
  • Array: An ordered map that can hold multiple values of different types. 
    $colors = ["Red", "Green", "Blue"];
$person = ["name" => "Bob", "age" => 30];
  • Object: An instance of a class.
  • NULL: A special data type representing a variable with no value. A variable can be set to NULL. 
    $var = null;
  • Resource: A special variable holding a reference to an external resource (e.g., a database connection, file handle).

You can use var_dump($variable) to display structured information about a variable, including its type and value. This is useful for debugging.

<?php
    $value = 10.5;
    var_dump($value); // Output: float(10.5)
?>

Operators

PHP supports various operators:

1. Arithmetic Operators

  • + (Addition), - (Subtraction), * (Multiplication), / (Division)
  • % (Modulus - remainder), ** (Exponentiation - PHP 5.6+)

2. Assignment Operators

  • = (Assign), +=, -=, *=, /=, %=
  • .= (String concatenation assignment) 
    $str = "Hello";
$str .= " World!"; // $str is now "Hello World!"

3. Comparison Operators

  • == (Equal), === (Identical - equal value and same type)
  • != or <> (Not equal), !== (Not identical)
  • > (Greater than), < (Less than), >=, <=
  • <=> (Spaceship operator - PHP 7+. Returns -1, 0, or 1 for less than, equal, or greater than)

4. Logical Operators

  • && or and (Logical AND)
  • || or or (Logical OR)
  • ! (Logical NOT)
  • xor (Logical XOR)

The and and or operators have lower precedence than && and ||. It's often safer to use && and || or parentheses to ensure correct order of operations.

5. Increment/Decrement Operators

  • ++$var (Pre-increment), $var++ (Post-increment)
  • --$var (Pre-decrement), $var-- (Post-decrement)

6. String Concatenation Operator

  • . (Dot) 
    $firstName = "John";
$lastName = "Doe";
$fullName = $firstName . " " . $lastName; // "John Doe"

7. Ternary Operator

(condition) ? value_if_true : value_if_false;

8. Null Coalescing Operator (PHP 7+)

$variable = $value_to_check ?? $default_value; (Assigns $value_to_check if it's not null, otherwise assigns $default_value).

Echo and Print

echo and print are language constructs (not functions, though they can be used with parentheses) used to output data to the browser/console.

  • echo:
    • Can output one or more strings.
    • Slightly faster than print (marginally).
    • Does not return a value.
    <?php
    echo "Hello", " ", "World!", "<br>";
    $name = "Alice";
    echo "My name is " . $name . "<br>";
    echo "This is a ", "test.";
?>
  • print:
    • Can only output one string.
    • Returns 1 (so it can be used in expressions, though rarely practical).
    <?php
    print "Hello from print!<br>";
    if (print("")) { // print returns 1, which evaluates to true
        // ...
    }
?>

For most cases, echo is preferred due to its slight performance edge and ability to take multiple arguments (though concatenation is more common for multiple items).

Control Flow

1. Conditional Statements

a) if, else if (or elseif), else

<?php
    $age = 20;
    if ($age < 18) {
        echo "You are a minor.<br>";
    } elseif ($age >= 18 && $age < 65) {
        echo "You are an adult.<br>"; // This will be printed
    } else {
        echo "You are a senior.<br>";
    }
?>

b) switch statement

<?php
    $favColor = "red";
    switch ($favColor) {
        case "red":
            echo "Your favorite color is red!<br>";
            break;
        case "blue":
            echo "Your favorite color is blue!<br>";
            break;
        default:
            echo "Your favorite color is neither red nor blue.<br>";
    }
?>

2. Looping Statements

a) for loop

<?php
    for ($i = 0; $i < 5; $i++) {
        echo "The number is: " . $i . "<br>";
    }
?>

b) while loop

<?php
    $count = 1;
    while ($count <= 3) {
        echo "While count is: " . $count . "<br>";
        $count++;
    }
?>

c) do...while loop

<?php
    $x = 6;
    do {
        echo "Do-while x is: " . $x . "<br>"; // Executes at least once
        $x++;
    } while ($x <= 5);
?>

d) foreach loop (for arrays and objects)

<?php
    $colors = ["red", "green", "blue"];
    foreach ($colors as $color) {
        echo $color . "<br>";
    }

    $person = ["name" => "Bob", "age" => 30];
    foreach ($person as $key => $value) {
        echo $key . ": " . $value . "<br>";
    }
?>

3. Loop Control Statements

  • break: Exits the current loop.
  • continue: Skips the rest of the current iteration and proceeds to the next.

Functions

A function is a block of reusable code that performs a specific task.

Defining and Calling Functions

<?php
    // Defining a function
    function greetUser($name) {
        echo "Hello, " . $name . "!<br>";
    }

    // Calling the function
    greetUser("Alice");
    greetUser("Bob");

    // Function with a return value
    function addNumbers($num1, $num2) {
        $sum = $num1 + $num2;
        return $sum;
    }

    $result = addNumbers(5, 7);
    echo "Sum: " . $result . "<br>";
?>

Function Arguments

  • Default Argument Values: 
    function showMessage($message = "Default message") {
    echo $message . "<br>";
}
showMessage(); // Outputs "Default message"
showMessage("Custom message");
  • Type Hinting (PHP 5+, enhanced in PHP 7+): You can specify the expected data type for parameters and return values. 
    // PHP 7+ return type declaration and scalar type hints
function multiply(int $a, int $b): int {
    return $a * $b;
}

Variable Scope

  • Local Scope: Variables declared inside a function are only accessible within that function.
  • Global Scope: Variables declared outside any function. To access a global variable inside a function, use the global keyword or the $GLOBALS array. 
    $globalVar = "I am global";
function myFunction() {
    global $globalVar;
    echo $globalVar; // Accesses global variable
    // OR echo $GLOBALS['globalVar'];
}
  • Static Scope: A static variable inside a function retains its value between function calls. 
    function counter() {
    static $count = 0;
    $count++;
    echo $count . "<br>";
}
counter(); // 1
counter(); // 2

Arrays

Arrays in PHP can store multiple values in one single variable.

Types of Arrays

  • Indexed Arrays: Arrays with numeric indices (starting from 0 by default). 
    <?php
    // Method 1:
    $cars = array("Volvo", "BMW", "Toyota");
    // Method 2 (short array syntax, PHP 5.4+):
    $fruits = ["Apple", "Banana", "Orange"];

    echo $cars[0]; // Volvo
    echo "<br>";
    $fruits[] = "Mango"; // Adds "Mango" to the end
    print_r($fruits); // print_r is useful for displaying arrays
?>
  • Associative Arrays: Arrays that use named keys (strings) that you assign to them. 
    <?php
    $age = ["Peter" => 35, "Ben" => 37, "Joe" => 43];
    echo "Peter is " . $age['Peter'] . " years old.<br>";

    $person = [
        'name' => 'Alice',
        'city' => 'New York'
    ];
    echo $person['city'];
?>
  • Multidimensional Arrays: Arrays containing other arrays. 
    <?php
    $students = [
        ["name" => "Alice", "grade" => "A"],
        ["name" => "Bob", "grade" => "B"],
        ["name" => "Charlie", "grade" => "A"]
    ];
    echo $students[1]['name']; // Bob
?>

Common Array Functions

PHP has a rich set of built-in array functions:
  • count($array): Returns the number of elements in an array.
  • sort($array) / rsort($array): Sorts indexed arrays (ascending/descending).
  • asort($array) / arsort($array): Sorts associative arrays by values (ascending/descending), preserving keys.
  • ksort($array) / krsort($array): Sorts associative arrays by keys (ascending/descending).
  • print_r($array) / var_dump($array): For displaying array contents for debugging.
  • array_push($array, $value1, ...): Adds elements to the end.
  • array_pop($array): Removes and returns the last element.
  • array_merge($array1, $array2): Merges arrays.
  • in_array($needle, $haystack): Checks if a value exists in an array.
  • array_key_exists($key, $array): Checks if a key exists in an array.

Superglobals

Superglobals are built-in variables that are always accessible, regardless of scope. You can access them from any function, class, or file without having to do anything special.

Common Superglobals:
  • $_GET: An associative array of variables passed to the current script via the URL parameters (query string).
    Example URL: script.php?name=Alice&age=30 
    $name = $_GET['name']; // "Alice"
$age = $_GET['age'];  // "30" (as string)
  • $_POST: An associative array of variables passed to the current script via the HTTP POST method (typically from HTML forms).
  • $_REQUEST: An associative array containing contents of $_GET, $_POST, and $_COOKIE. (Order depends on request_order in php.ini).
  • $_SERVER: An array containing information such as headers, paths, and script locations. 
    echo $_SERVER['PHP_SELF'];      // Filename of the currently executing script
echo $_SERVER['SERVER_NAME'];   // Name of the host server
echo $_SERVER['REQUEST_METHOD']; // Which request method was used (e.g., GET, POST)
  • $_SESSION: An associative array used to store session variables (data that persists across multiple pages for a user). Requires session_start().
  • $_COOKIE: An associative array of variables passed to the current script via HTTP Cookies.
  • $_FILES: An associative array of items uploaded to the current script via HTTP POST.
  • $GLOBALS: An associative array containing references to all variables which are currently defined in the global scope of the script.

When accessing data from superglobals like $_GET or $_POST, always validate and sanitize the input to prevent security vulnerabilities (e.g., XSS, SQL injection).

Form Handling

PHP is commonly used to process HTML form data submitted by users.

Example HTML Form (myform.html)

<!DOCTYPE html>
<html>
<head><title>PHP Form</title></head>
<body>
    <form action="process_form.php" method="post">
        Name: <input type="text" name="username"><br>
        Email: <input type="email" name="email"><br>
        <input type="submit" value="Submit">
    </form>
</body>
</html>

PHP Script to Process Form Data (process_form.php)

<?php
    if ($_SERVER["REQUEST_METHOD"] == "POST") {
        // Collect value of input fields
        // It's important to validate and sanitize user input!
        $name = isset($_POST['username']) ? htmlspecialchars($_POST['username']) : '';
        $email = isset($_POST['email']) ? filter_var($_POST['email'], FILTER_SANITIZE_EMAIL) : '';

        if (empty($name)) {
            echo "Name is empty<br>";
        } else {
            echo "Welcome, " . $name . "!<br>";
        }

        if (empty($email)) {
            echo "Email is empty<br>";
        } elseif (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
            echo "Invalid email format<br>";
        } else {
            echo "Your email is: " . $email . "<br>";
        }
    } else {
        echo "Form not submitted using POST method.";
    }
?>

Key functions used:

  • isset(): Checks if a variable is set and not null.
  • empty(): Checks if a variable is empty (e.g., "", 0, "0", null, false, array()).
  • htmlspecialchars(): Converts special characters to HTML entities (prevents XSS).
  • filter_var() with FILTER_SANITIZE_EMAIL and FILTER_VALIDATE_EMAIL: For sanitizing and validating email addresses.

Include and Require

You can include the content of one PHP file into another PHP file before the server executes it.

  • include 'filename.php';
    • If the file is not found, a warning is generated, but the script continues execution.
  • require 'filename.php';
    • If the file is not found, a fatal error is generated, and the script stops execution.
    • Use this for essential files (e.g., database connections, core functions).
  • include_once 'filename.php';
  • require_once 'filename.php';
    • These ensure that the file is included only once, even if the statement is encountered multiple times. This prevents errors from redeclaring functions or classes.

Example (header.php):

<?php echo "<header><h1>My Website Header</h1></header>"; ?>

Example (index.php):

<?php include_once 'header.php'; ?>
<p>Main content of the page.</p>
<?php include_once 'footer.php'; // Assuming footer.php exists ?>

Introduction to Object-Oriented Programming (OOP)

PHP has strong support for OOP, allowing you to structure your code using classes and objects.

Classes and Objects

<?php
    class Car {
        // Properties (member variables)
        public $color;
        public $model;

        // Constructor (called when an object is created)
        public function __construct($color, $model) {
            $this->color = $color;
            $this->model = $model;
            echo "A " . $this->color . " " . $this->model . " car is created.<br>";
        }
        
        // Destructor (called when an object is destroyed or script ends)
        public function __destruct() {
            echo "The " . $this->color . " " . $this->model . " car is destroyed.<br>";
        }

        // Method (member function)
        public function startEngine() {
            echo $this->model . " engine started! Vroom!<br>";
        }
    }

    // Create objects (instances of the Car class)
    $car1 = new Car("Red", "Ferrari");
    $car2 = new Car("Blue", "Toyota");

    $car1->startEngine();
    echo "Car 2 model: " . $car2->model . "<br>";
?>

Key OOP concepts like encapsulation (using public, private, protected access modifiers), inheritance (extends keyword), and polymorphism (abstract classes, interfaces, method overriding) are also supported in PHP.

Basic Error Handling

PHP has several ways to handle errors and exceptions.

  • die() or exit(): Terminates script execution and can output a message. 
    if (!file_exists("important_file.txt")) {
    die("Error: Important file not found. Script cannot continue.");
}
  • Error Reporting: The error_reporting() function sets which PHP errors are reported. In development, it's good to set it high (e.g., error_reporting(E_ALL); ini_set('display_errors', 1);). In production, errors should typically be logged to a file rather than displayed to users.
  • Custom Error Handlers: You can define your own functions to handle errors using set_error_handler().
  • Exception Handling (try-catch-finally): For more structured error handling, especially in OOP. 
    function divide($numerator, $denominator) {
    if ($denominator == 0) {
        throw new Exception("Division by zero!");
    }
    return $numerator / $denominator;
}

try {
    echo divide(10, 0);
} catch (Exception $e) {
    echo "Caught exception: " . $e->getMessage() . "<br>";
} finally {
    echo "Finally block executed.<br>"; // PHP 5.5+
}

Introduction to File Handling

PHP provides functions to read from and write to files on the server.

Reading a File

<?php
    $filename = "mytextfile.txt";
    // Ensure file exists and is readable
    if (file_exists($filename) && is_readable($filename)) {
        $file_content = file_get_contents($filename); // Reads entire file into a string
        echo "<pre>" . htmlspecialchars($file_content) . "</pre>";
    } else {
        echo "File does not exist or is not readable.<br>";
    }

    // Reading line by line (more memory efficient for large files)
    $file_handle = fopen($filename, "r"); // "r" for read mode
    if ($file_handle) {
        while (($line = fgets($file_handle)) !== false) {
            echo htmlspecialchars($line) . "<br>";
        }
        fclose($file_handle);
    }
?>

Writing to a File

<?php
    $output_filename = "output.txt";
    $data_to_write = "This is a line written by PHP.\nAnother line.\n";

    // Write entire string (overwrites file or creates if not exists)
    if (file_put_contents($output_filename, $data_to_write) !== false) {
        echo "Data written to " . $output_filename . " successfully.<br>";
    } else {
        echo "Error writing to file.<br>";
    }

    // Append to file
    $data_to_append = "Appending this line.\n";
    file_put_contents($output_filename, $data_to_append, FILE_APPEND);
?>

Functions like fopen(), fwrite(), `fgets()`, and fclose() provide more fine-grained control over file operations.

Next Steps

Congratulations on completing this beginner's guide to PHP! You've learned the core concepts needed to start building dynamic web applications.

To continue your PHP development journey:

  • Practice Regularly: Build small web applications, scripts, or experiment with the concepts you've learned.

    • CodeIgniter: A lightweight framework known for its speed and simplicity.
    • CakePHP, Laminas (formerly Zend Framework): Other established frameworks.
  • Content Management Systems (CMS):
    • WordPress: The most popular CMS, built with PHP. Learning to develop themes and plugins for WordPress can be a valuable skill.
    • Drupal, Joomla: Other powerful PHP-based CMS platforms.
  • Composer: Learn to use Composer, PHP's dependency manager, for managing project libraries.
  • REST APIs: Learn how to build and consume RESTful APIs with PHP.
  • Security Best Practices: Continuously learn about web security principles and how to write secure PHP code.
  • Version Control: Use Git for managing your code and collaborating.
  • Stay Updated: PHP is an evolving language. Follow official PHP news, blogs, and communities (like PHP.net, Laracasts, PHP The Right Way).

The world of PHP is vast. Focus on building a strong foundation, then explore areas that interest you most.

Happy PHP coding!
