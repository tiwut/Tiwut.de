A Comprehensive Beginner's Guide to CSS

Introduction to CSS

CSS stands for Cascading Style Sheets. It is a stylesheet language used to describe the presentation of a document written in a markup language like HTML. CSS describes how HTML elements should be rendered on screen, on paper, in speech, or on other media.

Key Roles of CSS:

  • Separation of Concerns: CSS allows you to separate the content structure (HTML) from its visual presentation. This makes code cleaner, easier to maintain, and more flexible.
  • Visual Styling: Controls colors, fonts, spacing, layout, and many other visual aspects of a web page.
  • Responsiveness: Enables the creation of web pages that adapt their layout to different screen sizes and devices (e.g., desktops, tablets, mobile phones).
  • Consistency: A single stylesheet can be used to style multiple web pages, ensuring a consistent look and feel across a website.

HTML provides the structure (the skeleton), while CSS provides the style (the clothes and appearance). They are fundamental technologies for building web pages, along with JavaScript for interactivity.

Ways to Use CSS

There are three primary ways to include CSS in an HTML document:

1. External Stylesheet

This is the most common and recommended method. Styles are defined in a separate .css file, which is then linked to the HTML document(s) using the <link> tag inside the <head> section.

Example HTML (index.html):

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Styled Page</title>
    <link rel="stylesheet" href="styles.css"> <!-- Link to external CSS file -->
</head>
<body>
    <h1>Hello, CSS!</h1>
    <p>This page is styled using an external stylesheet.</p>
</body>
</html>

Example CSS (styles.css):

body {
    font-family: Arial, sans-serif;
    background-color: #f0f0f0;
}

h1 {
    color: navy;
    text-align: center;
}

p {
    color: #333;
}

Advantages: Cleaner HTML, reusability across multiple pages, easier maintenance.

2. Internal Stylesheet (Embedded CSS)

CSS rules are placed directly within the HTML document inside a <style> tag, usually within the <head> section.

Example HTML:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Internal CSS Example</title>
    <style>
        body {
            font-family: Verdana, sans-serif;
            background-color: lightyellow;
        }
        h1 {
            color: green;
        }
    </style>
</head>
<body>
    <h1>Internal Styles</h1>
    <p>This page uses internal CSS.</p>
</body>
</html>

Use Cases: Useful for single HTML pages or when you need unique styles for a specific page that won't be reused.

3. Inline Styles

CSS rules are applied directly to individual HTML elements using the style attribute.

Example HTML:

<!DOCTYPE html>
<html lang="en">
<head>
    <title>Inline CSS Example</title>
</head>
<body>
    <h1 style="color: blue; text-align: right;">Inline Styled Heading</h1>
    <p style="font-size: 18px; color: darkred;">This paragraph has inline styles.</p>
</body>
</html>

Disadvantages: Mixes presentation with structure, makes maintenance difficult, has high specificity (can override other styles), and is generally discouraged for anything more than very specific, minor styling needs.

Basic CSS Syntax

A CSS rule consists of a selector and a declaration block.

  • Selector: Points to the HTML element(s) you want to style.
  • Declaration Block: Contains one or more declarations separated by semicolons (;). It is enclosed in curly braces {}.
  • Declaration: Consists of a CSS property name and a value, separated by a colon (:).

Structure:

selector {
    property1: value1;
    property2: value2;
    /* ... more declarations ... */
}

Example:

p { /* Selector targeting all <p> elements */
    color: red;          /* Declaration: property 'color', value 'red' */
    font-size: 16px;     /* Another declaration */
}

CSS Selectors

Selectors are patterns used to select the HTML element(s) you want to style.

Basic Selectors

  • Element Selector (Type Selector): Selects all elements of a specific type. 
    h2 { color: green; } /* Styles all <h2> elements */
  • Class Selector: Selects all elements with a specific class attribute. Preceded by a period (.). 
    <p class="highlight">This is highlighted.</p>
<div class="highlight">This is also highlighted.</div>
    .highlight { background-color: yellow; }
  • ID Selector: Selects a single element with a specific id attribute. Preceded by a hash (#). IDs must be unique within an HTML document. 
    <div id="main-header">Site Title</div>
    #main-header { font-size: 24px; font-weight: bold; }
  • Attribute Selector: Selects elements based on the presence or value of an attribute. 
    /* Selects all <input> elements with a type="text" attribute */
input[type="text"] { border: 1px solid #ccc; }

/* Selects all elements with a target="_blank" attribute */
[target="_blank"] { background-color: lightcyan; }
  • Universal Selector (*): Selects all HTML elements on the page. Use with caution as it can have performance implications and is often too broad. 
    * { margin: 0; padding: 0; box-sizing: border-box; } /* A common reset */

Grouping Selectors

To apply the same styles to multiple selectors, you can group them using commas (,).

h1, h2, h3 {
    font-family: 'Georgia', serif;
    color: #444;
}

Combinators

Combinators define the relationship between selectors.

  • Descendant Selector (space): Selects elements that are descendants of a specified element. 
    div p { color: blue; } /* Selects all <p> elements inside any <div> */
  • Child Selector (>): Selects elements that are direct children of a specified element. 
    ul > li { list-style-type: square; } /* Selects <li> that are direct children of <ul> */
  • Adjacent Sibling Selector (+): Selects an element that is immediately preceded by a specified element (they must share the same parent). 
    h2 + p { margin-top: 5px; } /* Selects a <p> immediately following an <h2> */
  • General Sibling Selector (~): Selects all elements that are siblings of a specified element and appear after it (they must share the same parent). 
    h2 ~ p { font-style: italic; } /* Selects all <p> siblings that come after an <h2> */

Comments in CSS

CSS comments are used to explain your code and can help when you edit the source code at a later date. Comments are ignored by browsers.

A CSS comment starts with /* and ends with */.

/* This is a single-line comment */
p {
    color: navy;
}

/*
This is a
multi-line comment.
It explains the following rule.
*/
.container {
    width: 90%;
    margin: 0 auto; /* Center the container */
}

Colors

CSS colors can be specified in several ways:

  • Named Colors: Predefined color names (e.g., red, blue, lightgray). There are about 140+ named colors. 
    body { background-color: lightcyan; }
h1 { color: darkblue; }
  • HEX (Hexadecimal) Colors: Specified as #RRGGBB, where RR, GG, BB are hexadecimal values (00-FF) for Red, Green, and Blue. A shorthand #RGB can be used if each pair is the same (e.g., #F00 is #FF0000). 
    .header { background-color: #336699; }
.title { color: #FFF; } /* White */
  • RGB Colors: Specified using the rgb(red, green, blue) function. Each parameter defines the intensity of the color (0-255). 
    p { color: rgb(255, 0, 0); } /* Red */
.subtle-text { color: rgb(100, 100, 100); }
  • RGBA Colors: An extension of RGB with an Alpha channel (opacity). rgba(red, green, blue, alpha). Alpha is a number between 0.0 (fully transparent) and 1.0 (fully opaque). 
    .overlay { background-color: rgba(0, 0, 0, 0.5); } /* Semi-transparent black */
  • HSL Colors: Hue, Saturation, Lightness. hsl(hue, saturation%, lightness%).
    • Hue: Degree on the color wheel (0-360). 0 is red, 120 is green, 240 is blue.
    • Saturation: Intensity of the color (0%-100%). 0% is gray, 100% is full color.
    • Lightness: Brightness (0%-100%). 0% is black, 50% is normal, 100% is white.
    .vibrant { color: hsl(120, 100%, 50%); } /* Pure green */
  • HSLA Colors: HSL with an Alpha channel. hsla(hue, saturation%, lightness%, alpha). 
    .translucent-blue { background-color: hsla(240, 100%, 50%, 0.7); }

Backgrounds

CSS background properties are used to define the background effects for elements.

  • background-color: Sets the background color of an element. 
    body { background-color: #EAEAEA; }
  • background-image: Sets an image as the background. 
    .hero { background-image: url("images/hero-banner.jpg"); }
  • background-repeat: Specifies if/how a background image will be repeated. Values: repeat (default), repeat-x, repeat-y, no-repeat. 
    .pattern { background-image: url("pattern.png"); background-repeat: repeat; }
  • background-position: Sets the starting position of a background image. Values like center, top left, percentages, or pixel values. 
    .logo-bg { background-image: url("logo.svg"); background-repeat: no-repeat; background-position: center top; }
  • background-attachment: Specifies whether a background image scrolls with the rest of the page or is fixed. Values: scroll (default), fixed. 
    body { background-image: url("sky.jpg"); background-attachment: fixed; }
  • background-size: Specifies the size of the background images. Values like auto, cover, contain, or specific dimensions. 
    .full-bg { background-image: url("large-image.jpg"); background-size: cover; background-position: center; }

Background Shorthand Property

You can set multiple background properties in one declaration using the background shorthand.

.hero-section {
    background: lightgray url("banner.png") no-repeat center center / cover;
    /* equivalent to:
    background-color: lightgray;
    background-image: url("banner.png");
    background-repeat: no-repeat;
    background-position: center center;
    background-size: cover; (Note: size needs / before it in shorthand if position is also set)
    */
}

Borders

CSS border properties allow you to specify the style, width, and color of an element's border.

  • border-width: Sets the width of the four borders. Can be thin, medium, thick, or a specific length (e.g., 5px). 
    div { border-width: 2px; }
  • border-style: Sets the style of the four borders. Common values: none, solid, dotted, dashed, double, groove, ridge, inset, outset. 
    p { border-style: solid; }
  • border-color: Sets the color of the four borders. 
    .box { border-color: #007bff; }

You can also specify properties for individual sides (e.g., border-top-style, border-left-width, border-bottom-color).

Border Shorthand Property

The border property is a shorthand for border-width, border-style, and border-color.

.container {
    border: 1px solid gray;
}

.alert {
    border-left: 5px solid red; /* Only a left border */
}

border-radius

Used to create rounded corners.

.rounded-box {
    border: 2px solid green;
    border-radius: 10px; /* All corners 10px radius */
}

.pill-button {
    border-radius: 50px; /* Often used for pill shapes if height is less than 100px */
    border-radius: 9999px; /* A common trick for fully rounded ends */
}

Margins and Padding

These properties are used to create space around and within elements, respectively. They are core to the CSS Box Model.

Padding

padding is the space between an element's content and its border. It clears an area around the content.

.padded-box {
    background-color: lightblue;
    border: 1px solid blue;
    padding: 20px; /* 20px padding on all four sides */
}

Padding properties can be set for individual sides: padding-top, padding-right, padding-bottom, padding-left.

Shorthand for padding:

  • padding: 20px; (all four sides)
  • padding: 10px 20px; (top/bottom 10px, left/right 20px)
  • padding: 5px 10px 15px; (top 5px, left/right 10px, bottom 15px)
  • padding: 5px 10px 15px 20px; (top, right, bottom, left - clockwise)

Margin

margin is the space outside the border of an element. It clears an area around the element, creating space between adjacent elements.

.spaced-box {
    background-color: lightgreen;
    border: 1px solid green;
    margin: 15px; /* 15px margin on all four sides */
    width: 200px;
}

Margin properties can be set for individual sides: margin-top, margin-right, margin-bottom, margin-left.

Shorthand for margin works the same way as for padding.

Centering Block Elements: To horizontally center a block-level element with a specified width, set its left and right margins to auto.

.centered-div {
    width: 50%;
    margin: 0 auto; /* 0 margin top/bottom, auto left/right */
    background-color: gold;
}

Margin Collapsing: Top and bottom margins of adjacent block-level elements can sometimes collapse into a single margin whose size is the largest of the individual margins. This doesn't happen for left and right margins.

Height and Width

The height and width properties are used to set the height and width of an element's content area.

div.box {
    height: 100px;
    width: 300px;
    background-color: coral;
}

Common units for height and width include:

  • px (pixels): Fixed size.
  • % (percentage): Relative to the containing block's width or height.
  • em: Relative to the font-size of the element.
  • rem: Relative to the font-size of the root (html) element.
  • vw (viewport width): 1vw = 1% of viewport width.
  • vh (viewport height): 1vh = 1% of viewport height.
  • auto: Default. Browser calculates height/width.

max-width and min-width

  • max-width: Sets the maximum width of an element. Useful for responsive design to prevent elements from becoming too wide on large screens while allowing them to shrink on smaller screens.
  • min-width: Sets the minimum width of an element.
.content-area {
    width: 80%;         /* Tries to be 80% of parent */
    max-width: 960px;    /* But not wider than 960px */
    min-width: 300px;    /* And not narrower than 300px */
    margin: 0 auto;
}

Similar properties max-height and min-height exist.

The CSS Box Model

All HTML elements can be considered as boxes. The CSS box model is a box that wraps around every HTML element. It consists of: margins, borders, padding, and the actual content.

(Image from MDN Web Docs)

  • Content: The actual content of the box, where text and images appear. Its dimensions are defined by width and height.
  • Padding: Clears an area around the content. The padding is transparent.
  • Border: A border that goes around the padding and content.
  • Margin: Clears an area outside the border. The margin is transparent.

When you set the width and height of an element, these properties define the width and height of the content area by default.

The total width of an element is: content width + left padding + right padding + left border + right border + left margin + right margin.

box-sizing Property

The box-sizing property allows you to change how the total width and height of an element are calculated.

  • content-box (default): The width and height properties apply only to the content of the element. Padding and border are added on top of this.
  • border-box: The width and height properties include the content, padding, and border, but not the margin. This often makes sizing elements more intuitive. 
    div {
    box-sizing: border-box;
    width: 300px;
    padding: 20px;
    border: 5px solid black;
    /* Total width of the element (excluding margin) will be 300px.
       Content area will be 300px - (2*20px padding) - (2*5px border) = 250px. */
}

Many developers prefer to set box-sizing: border-box; globally for all elements for easier layout management:

html {
  box-sizing: border-box;
}
*, *::before, *::after { /* Apply to all elements and pseudo-elements */
  box-sizing: inherit; /* Inherit from html element */
}

Text Styling

CSS provides many properties for formatting text.

  • color: Sets the color of the text. (Already covered in Colors section).
  • text-align: Specifies the horizontal alignment of text. Values: left, right, center, justify. 
    p.center { text-align: center; }
  • text-decoration: Adds decoration lines to text. Values: none, underline, overline, line-through. 
    a { text-decoration: none; } /* Common to remove underline from links */
.strikethrough { text-decoration: line-through; }
  • text-transform: Controls the capitalization of text. Values: none, capitalize, uppercase, lowercase. 
    h1 { text-transform: uppercase; }
  • line-height: Sets the distance between lines of text. Can use unitless numbers (multiplied by current font size), lengths, or percentages. 
    p { line-height: 1.6; } /* 1.6 times the normal line height */
  • letter-spacing: Increases or decreases the space between characters. 
    .wide-letters { letter-spacing: 2px; }
  • word-spacing: Increases or decreases the space between words. 
    .spaced-words { word-spacing: 10px; }
  • text-indent: Indents the first line of text in a block. 
    p.indent { text-indent: 30px; }
  • text-shadow: Adds shadow to text. text-shadow: h-shadow v-shadow blur-radius color; 
    h1.fancy { text-shadow: 2px 2px 5px gray; }

Fonts

CSS font properties define the typeface, size, weight, and style of text.

  • font-family: Specifies a prioritized list of font family names and/or generic family names. If the first font is not available, the browser tries the next one.
    • Generic families: serif (e.g., Times New Roman), sans-serif (e.g., Arial), monospace (e.g., Courier New), cursive, fantasy.
    body { font-family: "Helvetica Neue", Arial, sans-serif; }
code { font-family: "Consolas", "Lucida Console", monospace; }

    Font names containing spaces should be enclosed in quotes.

  • font-size: Sets the size of the font. Common units: px, em, rem, %, keywords (small, medium, large). 
    p { font-size: 16px; }
h1 { font-size: 2.5rem; }
  • font-weight: Sets the weight (boldness) of the font. Values: normal, bold, bolder, lighter, or numeric values (100-900). 
    .important-text { font-weight: bold; }
.sub-heading { font-weight: 700; } /* Equivalent to bold for many fonts */
  • font-style: Sets the style of the font. Values: normal, italic, oblique. 
    .emphasis { font-style: italic; }

Font Shorthand Property

The font property can be used as a shorthand for setting multiple font properties. Order matters: font-style font-variant font-weight font-size/line-height font-family. At least font-size and font-family are required.

body {
    font: italic bold 16px/1.5 Arial, sans-serif;
}

Web Fonts (@font-face)

Allows you to use fonts that are not installed on the user's computer by linking to font files on a web server.

@font-face {
  font-family: 'MyCustomFont';
  src: url('fonts/mycustomfont.woff2') format('woff2'),
       url('fonts/mycustomfont.woff') format('woff');
  font-weight: normal;
  font-style: normal;
}

body {
  font-family: 'MyCustomFont', sans-serif;
}

Alternatively, services like Google Fonts provide an easy way to use a wide variety of web fonts by linking to their stylesheet.

Styling Links

Links (<a> tags) can be styled differently depending on their state using pseudo-classes:

  • a:link: A normal, unvisited link.
  • a:visited: A link the user has visited.
  • a:hover: A link when the user mouses over it.
  • a:active: A link the moment it is clicked.

The order matters for these pseudo-classes (LVHA order: Link, Visited, Hover, Active) if you define all of them, due to specificity.

/* Unvisited link */
a:link {
  color: blue;
  text-decoration: underline;
}

/* Visited link */
a:visited {
  color: purple;
}

/* Mouse over link */
a:hover {
  color: red;
  text-decoration: none;
}

/* Selected link */
a:active {
  color: orange;
}

Styling Lists

CSS can be used to style HTML lists (<ul> and <ol>).

  • list-style-type: Changes the marker of list items.
    • For <ul>: none, disc (default), circle, square.
    • For <ol>: decimal (default), decimal-leading-zero, lower-roman, upper-roman, lower-alpha, upper-alpha, none.
    ul.checks { list-style-type: square; }
ol.letters { list-style-type: lower-alpha; }
  • list-style-image: Sets an image as the list item marker. 
    ul.custom-marker { list-style-image: url('custom-bullet.png'); }
  • list-style-position: Specifies whether the list-item markers should appear inside or outside the content flow. Values: inside, outside (default). 
    ul { list-style-position: inside; }
  • list-style: Shorthand property for all list style properties. 
    ul { list-style: circle inside url('marker.gif'); }

To completely remove default list styling (bullets/numbers and padding/margin), you often need:

ul.no-style {
    list-style-type: none;
    padding: 0;
    margin: 0;
}

The Display Property

The display property is one of the most important CSS properties for controlling layout. It specifies how an element is displayed.

Every HTML element has a default display value depending on its type.

Common Display Values

  • block:
    • Starts on a new line.
    • Takes up the full width available (stretches out to the left and right as far as it can).
    • Respects width, height, margin (top/bottom and left/right), and padding.
    • Examples: <div>, <h1>-<h6>, <p>, <ul>, <li>, <form>.
  • inline:
    • Does not start on a new line; flows along with surrounding text/content.
    • Only takes up as much width as necessary for its content.
    • Ignores width and height properties.
    • Vertical padding and margin (top/bottom) may not behave as expected or might overlap other elements. Horizontal padding and margin work.
    • Examples: <span>, <a>, <img> (though <img> is a bit special, often called a replaced inline element and can have width/height).
  • inline-block:
    • Flows like an inline element (doesn't start on a new line).
    • But, like a block element, it respects width, height, vertical/horizontal margin, and padding.
    span.button-like {
    display: inline-block;
    padding: 10px 15px;
    background-color: dodgerblue;
    color: white;
    border-radius: 5px;
}
  • none:
    • The element is completely removed from the page layout. It will not be visible, and it will not take up any space.
    .hidden { display: none; }

    (Contrast with visibility: hidden; which hides the element but still reserves its space in the layout).

Modern Layout Systems (Brief Introduction)

  • flex: Enables Flexbox layout, a one-dimensional layout model for arranging items in rows or columns. Excellent for component layout. 
    .flex-container { display: flex; justify-content: space-around; }
  • grid: Enables CSS Grid layout, a two-dimensional layout system for dividing a page into major regions or defining the relationship in terms of size, position, and layer between parts of a control built from HTML primitives. 
    .grid-container { display: grid; grid-template-columns: auto auto auto; }

Flexbox and Grid are powerful tools for modern web layout and are covered in more advanced CSS topics.

Positioning

The position property specifies the type of positioning method used for an element. It works in conjunction with the offset properties: top, right, bottom, left, and z-index.

  • static (default): Elements render in order, as they appear in the document flow. Offset properties have no effect.
  • relative: The element is positioned relative to its normal position. Setting offset properties will cause it to move from its normal spot, but the space it would have occupied in the normal flow is preserved. 
    .offset-box { position: relative; left: 20px; top: 10px; }
  • absolute: The element is positioned relative to its nearest positioned ancestor (an ancestor with a position value other than static). If no positioned ancestor exists, it is positioned relative to the initial containing block (usually the <body> or viewport). Absolute positioned elements are removed from the normal document flow. 
    .parent { position: relative; /* Establishes positioning context */ }
.child-absolute { position: absolute; top: 0; right: 0; width: 50px; }
  • fixed: The element is positioned relative to the browser viewport. It stays in the same place even if the page is scrolled. Fixed positioned elements are removed from the normal document flow. 
    .fixed-header { position: fixed; top: 0; left: 0; width: 100%; background-color: white; }
  • sticky: A hybrid of relative and fixed positioning. The element is treated as relatively positioned until it scrolls to a specified offset, at which point it becomes "stuck" (like fixed positioning) relative to its nearest scrolling ancestor or the viewport. 
    .sticky-nav { position: sticky; top: 0; background-color: #333; color: white; }

z-index

The z-index property specifies the stack order of a positioned element. An element with a greater stack order is always in front of an element with a lower stack order. z-index only works on positioned elements (position other than static).

.box1 { position: absolute; left: 10px; top: 10px; z-index: 1; background: red; }
.box2 { position: absolute; left: 20px; top: 20px; z-index: 2; background: blue; } /* box2 will be on top */

CSS Specificity

Specificity determines which CSS rule is applied by the browser if multiple rules target the same element and set the same property. More specific rules override less specific ones.

Specificity is usually calculated based on a combination of ID selectors, class selectors (or attribute/pseudo-class selectors), and element selectors.

General Order of Specificity (Highest to Lowest):

  1. Inline styles (e.g., <p style="color: red;">) - Highest specificity.
  2. ID selectors (e.g., #myElement)
  3. Class selectors (e.g., .myClass), attribute selectors (e.g., [type="text"]), and pseudo-classes (e.g., :hover)
  4. Element selectors (e.g., p) and pseudo-elements (e.g., ::before)
  5. Universal selector (*), inherited styles, and combinators have no specificity value themselves but contribute based on the selectors they combine.

If two selectors have the same specificity, the one that appears later in the CSS (or in a later stylesheet if multiple are linked) wins.

The !important rule can be added to a CSS property value to override any other declaration, regardless of specificity. However, using !important is generally discouraged as it makes debugging harder and breaks the natural cascade.

p { color: blue; }                          /* Specificity: 0,0,0,1 */
.content { color: green; }                   /* Specificity: 0,0,1,0 */
#intro { color: red; }                      /* Specificity: 0,1,0,0 */
p.content#intro { color: purple; }           /* Specificity: 0,1,1,1 (purple wins for <p class="content" id="intro">) */

.important-text { color: black !important; } /* Overrides others */

Understanding specificity is crucial for managing complex stylesheets and troubleshooting CSS conflicts.

CSS Units

CSS has various units for expressing lengths and sizes.

Absolute Units

Fixed and will appear as exactly that size, regardless of other elements or screen size. Not ideal for responsive design.

  • px (pixels): The most common absolute unit. 1px = 1/96th of an inch.
  • pt (points): 1pt = 1/72 of an inch. Often used for print.
  • cm, mm, in: Centimeters, millimeters, inches.

Relative Units

Relative to another length property. Better for responsive and accessible design.

  • em: Relative to the font-size of the parent element (if used for font-size itself) or the font-size of the element itself (for other properties like padding, margin). 
    .parent { font-size: 16px; }
.child { font-size: 1.2em; padding: 1em; } /* font-size: 19.2px, padding: 19.2px */
  • rem (root em): Relative to the font-size of the root (<html>) element. More predictable than em for font sizing across nested elements. 
    html { font-size: 10px; } /* Makes 1rem = 10px, for easier calculations */
h1 { font-size: 2.4rem; } /* 24px */
  • % (percentage): Relative to the value of the same property on the parent element (e.g., width: 50%; means 50% of parent's width). For font-size, it's relative to parent's font-size.
  • vw (viewport width): 1vw = 1% of the viewport's width.
  • vh (viewport height): 1vh = 1% of the viewport's height.
  • vmin: 1vmin = 1% of the viewport's smaller dimension (width or height).
  • vmax: 1vmax = 1% of the viewport's larger dimension.

Pseudo-classes and Pseudo-elements

Pseudo-classes

A pseudo-class is a keyword added to a selector that specifies a special state of the selected element(s). They start with a single colon (:).

Common pseudo-classes:

  • :link, :visited, :hover, :active: For styling link states (covered earlier).
  • :focus: Applies when an element (like an input field or button) has focus. 
    input:focus { border-color: dodgerblue; outline: none; }
  • :first-child: Selects the first child element of its parent.
  • :last-child: Selects the last child element of its parent.
  • :nth-child(n): Selects elements based on their position among siblings. n can be a number, keyword (odd, even), or a formula (2n+1). 
    li:nth-child(odd) { background-color: #f9f9f9; } /* Styles odd list items */
  • :not(selector): Selects elements that do not match the provided selector. 
    p:not(.special) { color: gray; } /* Styles paragraphs without class 'special' */

Pseudo-elements

A pseudo-element is a keyword added to a selector that lets you style a specific part of the selected element(s). They start with a double colon (::) in modern CSS (though single colon is often supported for backward compatibility for some).

Common pseudo-elements:

  • ::before: Inserts some content before the content of an element. Requires the content property.
  • ::after: Inserts some content after the content of an element. Requires the content property. 
    .important-note::before {
    content: "Note: ";
    font-weight: bold;
}
q::before { content: open-quote; }
q::after { content: close-quote; }
  • ::first-letter: Styles the first letter of a block-level element.
  • ::first-line: Styles the first line of a block-level element.
  • ::selection: Styles the portion of a document that has been highlighted by the user. 
    ::selection { background: gold; color: black; }

Introduction to Responsive Web Design

Responsive Web Design (RWD) is an approach to web design that makes web pages render well on a variety of devices and window or screen sizes. Content, design, and performance are necessary across all devices to ensure usability and satisfaction.

Key techniques for RWD include:

  • Fluid Grids: Using relative units like percentages for layout widths, rather than fixed pixel widths.
  • Flexible Images/Media: Using max-width: 100%; on images and other media to ensure they scale down within their container but don't exceed their original size. 
    img, video { max-width: 100%; height: auto; }
  • Media Queries: CSS3 module that allows content rendering to adapt to conditions such as screen resolution (e.g., smartphone vs. desktop).

Media Queries

Media queries use the @media rule to apply different styles for different media types/devices or specific characteristics (like viewport width).

/* Default styles for all screen sizes (often mobile-first) */
.container {
    width: 95%;
    margin: 0 auto;
}

.sidebar {
    display: none; /* Hide sidebar on small screens */
}

/* Styles for screens wider than 768px (e.g., tablets and desktops) */
@media screen and (min-width: 768px) {
    .container {
        width: 80%;
        max-width: 960px;
    }
    .sidebar {
        display: block; /* Show sidebar */
        float: right; /* Example old layout technique */
        width: 200px;
    }
}

/* Styles for screens wider than 1200px (e.g., large desktops) */
@media (min-width: 1200px) {
    body {
        font-size: 18px; /* Increase base font size */
    }
}

Remember to include the viewport meta tag in your HTML <head> for responsive design to work correctly:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Next Steps

Congratulations on learning the fundamentals of CSS! You now have the tools to style web pages.

To continue your CSS and web development journey:

  • Practice Extensively: Recreate layouts of websites you like. Experiment with different properties and selectors.
  • Master Layout Techniques:
    • Flexbox: For one-dimensional layouts (rows or columns). Essential for component layout.
    • CSS Grid: For two-dimensional page layouts (rows and columns simultaneously).
  • Deepen Responsive Design Skills: Practice mobile-first design, advanced media queries, and responsive images/typography.
  • Learn about CSS Variables (Custom Properties): For more maintainable and dynamic stylesheets. 
    :root { --main-color: blue; }
h1 { color: var(--main-color); }
  • Explore CSS Transitions and Animations: To add smooth visual effects and motion to your pages.
  • CSS Preprocessors: Learn Sass (SCSS) or Less. They add features like variables, nesting, mixins, and functions to CSS, making it more powerful and easier to manage large projects.
  • CSS Frameworks:
    • Bootstrap: A popular front-end framework with pre-styled components and a responsive grid system.
    • Tailwind CSS: A utility-first CSS framework that provides low-level utility classes to build custom designs directly in your HTML.
    • Materialize, Bulma, Foundation: Other notable frameworks.
  • Understand Browser Developer Tools: Become proficient in using your browser's dev tools (Inspector) to inspect elements, debug CSS, and experiment with styles live.
  • Web Performance: Learn how to write efficient CSS and optimize assets for faster page loads.
  • Accessibility (a11y): Ensure your CSS choices (e.g., color contrast, focus indicators) contribute to an accessible user experience.
  • Stay Updated: CSS is an evolving language. Follow blogs, MDN Web Docs, and other resources to learn about new features.

The combination of HTML, CSS, and JavaScript forms the core of front-end web development. Keep practicing and building!

Happy styling!
