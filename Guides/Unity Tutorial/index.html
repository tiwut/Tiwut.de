Unity Beginner's Guide: Create Your First Game

Introduction to Unity

Welcome to the Unity Beginner's Guide! Unity is a powerful and versatile game engine used by everyone from indie developers to large studios to create 2D and 3D games, as well as interactive experiences for a wide variety of platforms.

This guide will walk you through the basics, from installation to creating a simple interactive scene.

Note: Unity is constantly evolving. Screenshots and exact menu items may differ slightly in newer versions, but the core concepts remain the same.

1. Installation & Setup

The first step is to install Unity via the Unity Hub.

  1. Download Unity Hub: Go to the official Unity Download page and download the Unity Hub.
  2. Install Unity Hub: Run the downloaded installer and follow the instructions.
  3. Create a Unity ID: Start the Unity Hub. You will be prompted to sign in or create a Unity ID. This is free and necessary to use Unity.
  4. Install a Unity Editor Version:
    • In the Unity Hub, click on the "Installs" tab.
    • Click "Install Editor".
    • It's recommended to choose the latest LTS (Long-Term Support) version, as it's the most stable. Select it and click "Install".
    • During installation, you can select modules for different platforms (e.g., Android Build Support, iOS Build Support, WebGL Build Support). For starters, the default installation for desktop (Windows/Mac/Linux) is sufficient. You can always add modules later.
    • The download and installation can take some time.

2. Understanding Unity Hub

The Unity Hub is your central point for managing your Unity projects and installations.

  • Projects: Here you can see and open your created projects or create new ones.
  • Installs: Manage your installed Unity Editor versions. You can have multiple versions installed side-by-side.
  • Learn: Access tutorials and learning resources from Unity.
  • Community: Links to forums, Unity Answers, and other community platforms.

It's good practice to use the Editor version a project was created with for that specific project to avoid compatibility issues.

3. The Unity Editor: An Overview

After creating a new project in Unity Hub (e.g., select the "3D Core" template), the Unity Editor will open. The main windows are:

https://learn.unity.com/tutorial/explore-the-unity-editor-1#
  • Scene View: Your visual workspace for placing and manipulating GameObjects in your 3D or 2D world.
    • Navigation: Hold Right Mouse Button + WASD to fly, Mouse Wheel to zoom, Middle Mouse Button to pan.
  • Game View: Shows what the camera in the game sees. This is where you test your game. Click the "Play" button at the top center to start the game.
  • Hierarchy Window: Lists all GameObjects in the current scene. Objects can be nested hierarchically (parent-child relationships).
  • Project Window: Your file explorer for the project. Here you'll find all assets like models, textures, scripts, audio files, etc. The important folder is Assets.
  • Inspector Window: Displays the properties and components of the currently selected GameObject or asset. This is where you'll make most adjustments.
  • Console Window: Shows error messages, warnings, and debug outputs from your scripts (Window > General > Console).

You can customize the layout of the windows by dragging and dropping their tabs. Under Window > Layouts, you can choose from preset layouts or save your own.

4. GameObjects & Components

The core concept in Unity involves GameObjects and Components.

  • GameObject: Think of a GameObject as an empty container in your scene. It represents anything that exists in your game: characters, lights, cameras, props, etc. An empty GameObject does nothing on its own.
  • Components: Components give GameObjects functionality and behavior. Every GameObject has at least one component: the Transform component, which determines its position, rotation, and scale in the world.

Examples of components:

  • Mesh Renderer: Makes a 3D model visible.
  • Light: Makes a GameObject emit light.
  • Camera: Defines the player's viewpoint.
  • Rigidbody: Enables physics behavior (gravity, collisions).
  • Audio Source: Allows a GameObject to play sound.
  • Script: Custom logic you write in C#.

To create a GameObject: Right-click in the Hierarchy window and select, for example, 3D Object > Cube. Select the Cube and look at its components in the Inspector.

5. Assets & Prefabs

Assets

Assets are all the files you use in your project. These include:

  • 3D Models (e.g., .fbx, .obj)
  • Textures (images, e.g., .png, .jpg)
  • Materials (define the appearance of surfaces)
  • Audio Files (.wav, .mp3)
  • Scripts (.cs C# files)
  • Scenes (.unity files, save the state of your levels)
  • Animations

All assets are organized in the Assets folder of your Project window. It's advisable to create a clear folder structure here (e.g., folders for Scripts, Materials, Textures, Models, etc.).

Prefabs

A Prefab is a template for a GameObject, including all its components and settings. If you need a GameObject часто (e.g., an enemy, a coin, a tree), you can save it as a Prefab.

  1. Create and configure a GameObject in your scene as desired.
  2. Drag this GameObject from the Hierarchy window into a folder in the Project window. It will be saved there as a Prefab file (blue icon).
  3. You can now drag as many instances of this Prefab into your scene as you like.

The big advantage: If you change the Prefab in the Project window (e.g., adjust a material or add a script), all instances of that Prefab in your scene will be automatically updated.

6. Basic C# Scripting

Unity uses the C# (C-Sharp) programming language for game logic. Scripts are components that you add to GameObjects.

Creating and Attaching a Script:

  1. In the Project window: Right-click > Create > C# Script. Name it, for example, PlayerMovement.
  2. Select a GameObject in your scene (e.g., the Cube you created earlier).
  3. Drag the PlayerMovement script from the Project window onto the selected GameObject in the Inspector or directly onto the object in the Hierarchy. Alternatively, you can click "Add Component" in the Inspector and search for the script name.

Basic Structure of a Unity C# Script:

Open the script by double-clicking it. It will open in your default code editor (often Visual Studio or VS Code).


using UnityEngine; // Important for accessing Unity functions

public class PlayerMovement : MonoBehaviour
{
    // Variables are declared here (outside of functions)
    public float speed = 5.0f; // 'public' makes the variable visible in the Inspector

    // Start() is called once when the game starts,
    // after the object is enabled.
    void Start()
    {
        Debug.Log("PlayerMovement script started!");
        // Example: Change the GameObject's position
        // transform is a reference to the Transform component of this GameObject
        transform.position = new Vector3(0, 1, 0); 
    }

    // Update() is called once per frame.
    // Most game logic goes here (e.g., input polling, movement).
    void Update()
    {
        // Example: Move the object to the right
        // Time.deltaTime ensures frame-rate independent movement
        transform.Translate(Vector3.right * speed * Time.deltaTime); 

        // Example: Polling for input
        if (Input.GetKeyDown(KeyCode.Space))
        {
            Debug.Log("Spacebar pressed!");
        }
    }
}
  • using UnityEngine;: Imports the Unity namespace, which allows access to classes like MonoBehaviour, GameObject, Transform, Debug, Input, etc.
  • public class YourScriptName : MonoBehaviour: Your class inherits from MonoBehaviour. Only then can it be attached as a component to a GameObject and have access to Unity-specific functions like Start() and Update().
  • Start(): Called once when the script is enabled, typically at the beginning of the game. Good for initializations.
  • Update(): Called every frame. This is where you implement logic that needs to be executed continuously (e.g., player movement, state checks).
  • public float speed = 5.0f;: A public variable. The value 5.0f is the default. You can change this value directly in the Inspector window for the GameObject the script is attached to, without recompiling the code! The f stands for float.
  • transform: A built-in shortcut to the Transform component of the GameObject this script is attached to. Allows access to position, rotation, and scale.
  • Debug.Log("Message");: Prints a message to the Console window. Very useful for debugging.
  • Input.GetKeyDown(KeyCode.Space): Checks if the spacebar was pressed down in this frame.

Save your script in the code editor (Ctrl+S or Cmd+S). Unity will then automatically compile it in the background.

7. Creating Your First Scene

Let's create a simple scene with a moving cube.

  1. New Scene (optional): If you're not in an empty scene, go to File > New Scene (select "Basic (Built-in)" or "Empty"). Save the scene via File > Save As... (e.g., as MainScene.unity in the Assets/Scenes folder).
  2. Create a Floor:
    • Right-click in Hierarchy > 3D Object > Plane.
    • Select the Plane. In the Inspector under Transform, set Position to (0, 0, 0) and Scale to, for example, (5, 1, 5) to make it larger.
  3. Create a Player Cube:
    • Right-click in Hierarchy > 3D Object > Cube.
    • Rename the Cube in the Inspector to "Player".
    • Set its Position in the Inspector to (0, 0.5, 0) so it hovers slightly above the Plane.
  4. Create a Movement Script for the Player:
    • Create a new C# Script named SimpleMover in the Project window (e.g., in the Assets/Scripts folder).
    • Paste the following code into the script: 
      
using UnityEngine;

public class SimpleMover : MonoBehaviour
{
    public float moveSpeed = 5f;
    public float turnSpeed = 180f; // Degrees per second

    void Update()
    {
        // Forward/Backward movement based on Up/Down arrow keys or W/S
        float verticalInput = Input.GetAxis("Vertical"); // Values between -1 and 1
        transform.Translate(Vector3.forward * verticalInput * moveSpeed * Time.deltaTime);

        // Rotation based on Left/Right arrow keys or A/D
        float horizontalInput = Input.GetAxis("Horizontal"); // Values between -1 and 1
        transform.Rotate(Vector3.up, horizontalInput * turnSpeed * Time.deltaTime);
    }
}
    • Save the script.
    • Drag the SimpleMover script onto your "Player" GameObject in the Hierarchy or Inspector.
    • Select the "Player". You should now see the "Simple Mover (Script)" component in the Inspector, with public variables Move Speed and Turn Speed that you can adjust.
  5. Adjust Camera (optional): Select the "Main Camera" in the Hierarchy. Position it so it has a good view of the Player and Plane (e.g., Position (0, 3, -5) and Rotation X to about 20-30 degrees). You can see the preview in the Game window.
  6. Test: Click the Play button at the top. You should now be able to control the cube with the arrow keys or WASD! Click Play again to exit Play mode.

Changes you make in the Inspector during Play mode will not be saved when you exit Play mode! Always make important changes outside of Play mode.

8. Basic Physics

Unity has a built-in physics system (based on Nvidia PhysX) that enables realistic interactions.

Rigidbody

For a GameObject to be affected by the physics engine (gravity, forces, collisions), it needs a Rigidbody component.

  1. Select your "Player" cube.
  2. In the Inspector, click "Add Component".
  3. Search for "Rigidbody" and add it.

Now, if you start the game, the cube will fall onto the plane due to gravity. The default Rigidbody is for 3D physics. For 2D games, there's Rigidbody 2D.

Collider

For GameObjects to collide with each other, they need Collider components. Colliders define the physical shape of an object for collision calculations. They don't have to exactly match the visible shape (simpler collider shapes are often more performant).

  • Primitive objects like Cubes, Spheres, and Planes automatically get matching colliders (Box Collider, Sphere Collider, Mesh Collider for the Plane).
  • You can manually add/adjust colliders via "Add Component" > "Physics" > (e.g., Box Collider, Sphere Collider, Capsule Collider, Mesh Collider).
  • Ensure your Plane also has a collider (it should be a Mesh Collider by default) so the cube doesn't fall through.
  • Is Trigger: If this checkbox is enabled on a collider, it will fire collision events (e.g., OnTriggerEnter in a script), but there will be no physical collision response (objects can pass through each other). Useful for collectibles or zones.

At least one of the colliding GameObjects with a collider must also have a Rigidbody for physical collisions (pushing, bouncing) to occur. Two static colliders (without a Rigidbody) will not physically interact with each other.

9. UI Basics

Unity UI is used to create user interfaces like menus, health bars, score displays, etc.

  1. Create a Canvas:
    • Right-click in Hierarchy > UI > Canvas. A Canvas GameObject will be created, along with an EventSystem (important for interactions like button clicks).
    • The Canvas is the root container for all UI elements.
    • In the Canvas's Inspector, set the "Render Mode" often to "Screen Space - Overlay" (UI is drawn on top of everything else) or "Screen Space - Camera" (UI is rendered by a specific camera, useful for 3D effects on UI).
    • The "Canvas Scaler" (a component on the Canvas) helps adapt the UI to different screen resolutions. A common setting is "Scale With Screen Size" with a reference resolution (e.g., 1920x1080).
  2. Add a Text Element:
    • Right-click on the Canvas GameObject in the Hierarchy > UI > Text - TextMeshPro. (If TextMeshPro is used for the first time, you might need to confirm "Import TMP Essentials").
    • Position the Text element in the Scene View (make sure you are in 2D mode in the Scene View to edit UI more easily - top toolbar of Scene View).
    • In the Text element's Inspector, you can adjust text, font, color, size, etc.
  3. Add a Button:
    • Right-click on the Canvas > UI > Button - TextMeshPro.
    • A Button typically has a child GameObject with a Text component for its label.
    • The functionality of a Button is defined via the "On Click ()" event in the Button component's Inspector. You can call public functions from scripts here.

Example: Button Click Increments Score

  1. Create a C# Script named UIManager. 
    
using UnityEngine;
using TMPro; // Important for TextMeshPro

public class UIManager : MonoBehaviour
{
    public TextMeshProUGUI scoreText; // Drag your Text element here in the Inspector
    private int score = 0;

    void Start()
    {
        UpdateScoreText();
    }

    public void IncrementScore() // We'll call this function from the button
    {
        score++;
        UpdateScoreText();
    }

    void UpdateScoreText()
    {
        if (scoreText != null)
        {
            scoreText.text = "Score: " + score;
        }
    }
}
  2. Create an empty GameObject in the scene (Right-click Hierarchy > Create Empty), name it "GameManager" and add the UIManager script to it.
  3. Select the GameManager. Drag your TextMeshPro Text element from the Hierarchy into the "Score Text" field in the UIManager script's Inspector.
  4. Select your Button. In the Inspector, under the Button component, find the "On Click ()" section.
    • Click the + sign.
    • Drag your "GameManager" GameObject from the Hierarchy into the field that says "None (Object)".
    • Click the "No Function" dropdown menu and select UIManager > IncrementScore().
  5. Start the game. Each click on the button should now increase the score in the text field.

10. Building Your Game

When you want to share your game with others or run it standalone, you need to "build" it.

  1. Go to File > Build Settings....
  2. Add Scenes: In the "Scenes In Build" section, click "Add Open Scenes" to add your currently open scene(s). Ensure the starting scene is at the top (index 0) and checked.
  3. Choose Platform: On the left, select the target platform (e.g., "PC, Mac & Linux Standalone", "WebGL", "Android").
    • If the platform isn't installed, there's usually an "Install with Unity Hub" button.
    • For "PC, Mac & Linux Standalone": Select the "Target Platform" (Windows, macOS, Linux).
  4. Player Settings (Optional but important): Click the "Player Settings..." button at the bottom left. Here you can configure many specific settings for your game:
    • Company Name, Product Name, Version
    • Default Icon, Cursors
    • Resolution and Presentation (windowed mode, supported resolutions)
    • Splash Image (the image displayed when the game starts)
    • Many platform-specific settings.
  5. Create Build:
    • If you click "Build", you'll be prompted to choose a folder where the game files will be saved. It's best to create a new, empty folder (e.g., "Builds" outside your Assets folder).
    • If you click "Build And Run", the game will be built and then started immediately.

The build process can take some time depending on the project size and complexity.
