A Comprehensive Beginner's Guide to Java Programming

Introduction to Java

Java is a high-level, class-based, object-oriented programming language that is designed to have as few implementation dependencies as possible. It is a general-purpose programming language intended to let application developers "write once, run anywhere" (WORA), meaning that compiled Java code can run on all platforms that support Java without the need for recompilation.

Key Features of Java:

  • Object-Oriented: In Java, everything is an Object (or can be treated as one).
  • Platform Independent: Java code is compiled into bytecode, which can be run on any machine that has a Java Virtual Machine (JVM) installed.
  • Statically-Typed: Variable types are checked at compile-time, leading to more robust code.
  • Robust and Secure: Java has features like automatic memory management (garbage collection) and exception handling.
  • Multithreaded: Java supports multithreaded programming, which allows concurrent execution of two or more parts of a program.
  • Rich API: Java provides a large standard library with a wide range of functionalities.

Java is widely used for developing enterprise-level applications, Android mobile apps, web applications, big data technologies, scientific applications, and more.

Setting Up Your Environment

To start programming in Java, you need to install the Java Development Kit (JDK).

1. Installing the JDK

  1. Download the JDK: You can download the JDK from Oracle (official, may require an account for older versions or specific licenses) or opt for an OpenJDK distribution like Adoptium (Temurin), Amazon Corretto, or Azul Zulu. For beginners, Adoptium's Temurin is a good choice.
    • Oracle JDK: oracle.com/java/technologies/downloads/
    • Adoptium (Temurin): adoptium.net
  2. Choose a Version: LTS (Long-Term Support) versions like Java 11, 17, or 21 are generally recommended for stability. The latest non-LTS version is also fine for learning.
  3. Install the JDK: Run the installer and follow the on-screen instructions.

    During installation, or afterwards, ensure that the JDK's bin directory is added to your system's PATH environment variable. This allows you to run Java commands from any directory in your terminal. Many installers offer an option to do this automatically.

  4. Verify Installation: Open your terminal or command prompt and type: 
    java -version
    And also: 
    javac -version
    You should see the installed Java version printed for both commands. java is the command to run Java applications, and javac is the Java compiler.

2. Choosing an Integrated Development Environment (IDE)

An IDE provides tools like a code editor, debugger, and build automation tools, which greatly simplify Java development.

Popular choices for Java:
  • IntelliJ IDEA: Very popular and powerful. Has a free Community Edition and a paid Ultimate Edition. Download IntelliJ IDEA
  • Eclipse IDE for Java Developers: A long-standing, free, and open-source IDE. Download Eclipse
  • Visual Studio Code (VS Code) with Java Extensions: Lightweight editor with excellent Java support through extensions like "Extension Pack for Java" from Microsoft. Download VS Code

For this guide, examples will be simple enough to compile and run from the terminal, but using an IDE is highly recommended for any real development.

Your First Java Program: "Hello, World!"

Let's create a simple program that prints "Hello, World!" to the console.

  1. Create a file: Open your text editor or IDE and create a new file named HelloWorld.java.

    In Java, the file name must match the public class name defined within the file (case-sensitive).

  2. Write the code: 
    public class HelloWorld {
    // The main method is the entry point of any Java application
    public static void main(String[] args) {
        // Print "Hello, World!" to the console
        System.out.println("Hello, World!");
    }
}
  3. Save the file.
  4. Compile the code: Open your terminal or command prompt, navigate to the directory where you saved HelloWorld.java, and type: 
    javac HelloWorld.java
    If there are no errors, this command will create a new file named HelloWorld.class. This file contains the Java bytecode.
  5. Run the program: In the same terminal, type: 
    java HelloWorld
    (Note: you do not include the .class extension when running).

You should see the output:

Hello, World!

Explanation:

  • public class HelloWorld: Declares a public class named HelloWorld. All Java code resides within classes.
  • public static void main(String[] args): This is the main method.
    • public: It's an access modifier, meaning the method is accessible from any other class.
    • static: Allows the main method to be called without creating an object of the class.
    • void: Indicates that the main method does not return any value.
    • main: This is the default signature recognized by the JVM as the starting point of the program.
    • String[] args: An array of strings that can be used to pass command-line arguments to the program.
  • System.out.println("Hello, World!");: This line prints the string "Hello, World!" followed by a newline to the console.
    • System: A built-in class that provides access to system resources.
    • out: A static member of the System class; it's an instance of PrintStream.
    • println(): A method of PrintStream that prints the argument passed to it.
  • Curly braces {} define blocks of code (e.g., class body, method body).
  • Semicolons ; terminate statements in Java.

Variables and Data Types

Variables are named memory locations that store data. In Java, you must declare the type of a variable before you can use it (statically-typed).

Declaring and Initializing Variables

// Declaration: type variableName;
// Initialization: variableName = value;
// Declaration and Initialization: type variableName = value;

int age = 30;             // Integer type
double salary = 50000.75;  // Double-precision floating-point type
char grade = 'A';          // Character type (single quotes)
boolean isEmployed = true; // Boolean type (true or false)
String name = "John Doe";  // String type (a sequence of characters, a class, not a primitive)

System.out.println("Name: " + name);
System.out.println("Age: " + age);
System.out.println("Is Employed: " + isEmployed);

Primitive Data Types

Java has eight primitive data types:

  • byte: 8-bit signed integer. Range: -128 to 127. 
    byte smallNumber = 100;
  • short: 16-bit signed integer. Range: -32,768 to 32,767. 
    short mediumNumber = 20000;
  • int: 32-bit signed integer. Range: -231 to 231-1. (Most commonly used integer type). 
    int population = 1000000;
  • long: 64-bit signed integer. Range: -263 to 263-1. Use when int is not large enough. Suffix L or l. 
    long veryLargeNumber = 123456789012L;
  • float: 32-bit single-precision floating-point number. Suffix F or f. 
    float price = 19.99F;
  • double: 64-bit double-precision floating-point number. (Default type for decimal numbers). 
    double piValue = 3.1415926535;
  • boolean: Represents two values: true or false. 
    boolean isActive = false;
  • char: 16-bit Unicode character. Enclosed in single quotes. 
    char initial = 'J';

Reference Data Types (Non-Primitive)

Reference types refer to objects. They store the memory address of an object.

  • String: A sequence of characters. Strings are objects in Java, not primitive types. 
    String greeting = "Hello, Java!";
  • Arrays: Collections of elements of the same type.
  • Classes, Interfaces, Enums: User-defined types.

The default value for uninitialized primitive numeric types is 0 (or 0.0), for boolean is false, for char is '\u0000', and for reference types is null.

Type Casting

Converting a value from one data type to another.

  • Widening Casting (Implicit): Smaller type to a larger type size. Done automatically. 
    int myInt = 9;
double myDouble = myInt; // Automatic casting: int to double
System.out.println(myDouble); // Output: 9.0
  • Narrowing Casting (Explicit): Larger type to a smaller size type. Must be done manually by placing the type in parentheses. Risk of data loss. 
    double preciseDouble = 9.78;
int castedInt = (int) preciseDouble; // Manual casting: double to int
System.out.println(castedInt); // Output: 9 (fractional part lost)

Operators

Operators perform operations on variables and values.

1. Arithmetic Operators

  • + (Addition), - (Subtraction), * (Multiplication), / (Division), % (Modulus - remainder)
  • ++ (Increment: increases value by 1), -- (Decrement: decreases value by 1)
int a = 10, b = 3;
System.out.println("a + b = " + (a + b)); // 13
System.out.println("a / b = " + (a / b)); // 3 (integer division)
System.out.println("a % b = " + (a % b)); // 1

int c = 5;
c++; // c becomes 6
System.out.println("c = " + c);

2. Relational (Comparison) Operators

Return true or false.

  • == (Equal to), != (Not equal to), > (Greater than), < (Less than), >= (Greater than or equal to), <= (Less than or equal to)
int x = 5, y = 10;
System.out.println("x == y: " + (x == y)); // false
System.out.println("x < y: " + (x < y));   // true

3. Logical Operators

  • && (Logical AND), || (Logical OR), ! (Logical NOT)
boolean condition1 = true;
boolean condition2 = false;
System.out.println("condition1 && condition2: " + (condition1 && condition2)); // false
System.out.println("!condition2: " + (!condition2)); // true

4. Assignment Operators

  • =, +=, -=, *=, /=, %=
int score = 0;
score += 10; // score is now 10 (same as score = score + 10)
System.out.println("Score: " + score);

5. Ternary Operator

A shorthand for an if-else statement.

variable = (condition) ? expressionTrue : expressionFalse;

int time = 20;
String result = (time < 18) ? "Good day." : "Good evening.";
System.out.println(result); // Output: Good evening.

Input and Output

Output

We've already seen System.out.println() for printing output followed by a newline.

  • System.out.print(): Prints output without a newline.
  • System.out.printf(): Prints formatted output (similar to C's printf).
System.out.print("Hello, ");
System.out.print("Java!\n"); // \n for newline

String name = "Alice";
int age = 25;
System.out.printf("Name: %s, Age: %d\n", name, age); // %s for string, %d for integer

Input using Scanner

The Scanner class (from java.util package) is used to get user input.

import java.util.Scanner; // Import the Scanner class

public class UserInput {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in); // Create a Scanner object

        System.out.print("Enter your name: ");
        String userName = scanner.nextLine();  // Read a full line of text

        System.out.print("Enter your age: ");
        int userAge = scanner.nextInt();    // Read an integer

        System.out.print("Enter your salary: ");
        double userSalary = scanner.nextDouble(); // Read a double

        System.out.println("\nUser Details:");
        System.out.println("Name: " + userName);
        System.out.println("Age: " + userAge);
        System.out.println("Salary: " + userSalary);

        scanner.close(); // Close the scanner to release resources
    }
}

After reading a number with nextInt(), nextDouble(), etc., if you then try to read a line with nextLine(), the nextLine() might consume the leftover newline character from the previous input. A common fix is to add an extra scanner.nextLine(); call after reading a number and before reading a line.

Control Flow

Control flow statements dictate the order in which statements are executed.

1. Conditional Statements

a) if-else if-else

int score = 85;
char grade;

if (score >= 90) {
    grade = 'A';
} else if (score >= 80) {
    grade = 'B';
} else if (score >= 70) {
    grade = 'C';
} else {
    grade = 'D';
}
System.out.println("Grade: " + grade); // Output: Grade: B

b) switch statement

Used for selecting one of many code blocks to be executed. Works with byte, short, char, int, String (since Java 7), and enums.

int day = 4;
String dayName;

switch (day) {
    case 1:
        dayName = "Monday";
        break;
    case 2:
        dayName = "Tuesday";
        break;
    case 3:
        dayName = "Wednesday";
        break;
    case 4:
        dayName = "Thursday";
        break;
    // ... other cases
    default:
        dayName = "Invalid day";
        break; // break in default is optional if it's the last statement
}
System.out.println(dayName); // Output: Thursday

Don't forget the break statement in each case. Without it, execution "falls through" to the next case.

2. Looping Statements

a) for loop

// Traditional for loop
for (int i = 0; i < 5; i++) { // (initialization; condition; update)
    System.out.println("Iteration: " + i);
}
// Output: Iteration: 0 to Iteration: 4

b) Enhanced for loop (for-each loop)

Used for iterating over arrays or collections.

String[] fruits = {"Apple", "Banana", "Cherry"};
for (String fruit : fruits) {
    System.out.println(fruit);
}
// Output: Apple, Banana, Cherry

c) while loop

int count = 0;
while (count < 3) {
    System.out.println("Count is: " + count);
    count++;
}
// Output: Count is: 0 to Count is: 2

d) do-while loop

Executes the code block at least once, then checks the condition.

int num = 5;
do {
    System.out.println("Number (do-while): " + num);
    num++;
} while (num < 5); // Condition is false initially, but loop runs once
// Output: Number (do-while): 5

3. Loop Control Statements

  • break: Exits the loop.
  • continue: Skips the current iteration and proceeds to the next.
for (int i = 0; i < 10; i++) {
    if (i == 4) {
        continue; // Skip iteration when i is 4
    }
    if (i == 7) {
        break;    // Exit loop when i is 7
    }
    System.out.print(i + " "); // Output: 0 1 2 3 5 6
}
System.out.println();

Methods

A method is a block of code that performs a specific task. It runs only when it is called. You can pass data, known as parameters, into a method.

Defining and Calling Methods

public class MethodExample {

    // Method without parameters and no return value
    public static void greet() {
        System.out.println("Hello from the greet method!");
    }

    // Method with parameters and a return value
    public static int add(int num1, int num2) {
        int sum = num1 + num2;
        return sum; // Returns the sum
    }

    // Method with parameters and no return value
    public static void printInfo(String name, int age) {
        System.out.println("Name: " + name + ", Age: " + age);
    }

    public static void main(String[] args) {
        greet(); // Calling the greet method

        int result = add(5, 3); // Calling add and storing the returned value
        System.out.println("Sum: " + result); // Output: Sum: 8

        printInfo("Alice", 30); // Calling printInfo
    }
}

Method Signature: Consists of the method name and the parameter list (number, type, and order of parameters).

Return Type: The data type of the value returned by the method. Use void if the method does not return a value.

Access Modifiers: (e.g., public, private) control visibility. static means the method belongs to the class, not an object instance.

Method Overloading

Allows multiple methods to have the same name but different parameter lists (different number of parameters, or different types of parameters, or different order).

public class OverloadingExample {
    public static int add(int a, int b) {
        return a + b;
    }

    public static double add(double a, double b) {
        return a + b;
    }

    public static int add(int a, int b, int c) {
        return a + b + c;
    }

    public static void main(String[] args) {
        System.out.println(add(5, 10));        // Calls first add method
        System.out.println(add(3.5, 2.5));     // Calls second add method
        System.out.println(add(1, 2, 3));      // Calls third add method
    }
}

Arrays

Arrays are used to store multiple values of the same type in a single variable. The size of an array is fixed once it's created.

Declaring and Initializing Arrays

// Declaration
int[] numbers;
String[] names;

// Instantiation (allocating memory)
numbers = new int[5]; // An array of 5 integers (indices 0-4)
names = new String[3]; // An array of 3 Strings

// Initialization
numbers[0] = 10;
numbers[1] = 20;
// ... numbers[4] = 50;

// Declaration, Instantiation, and Initialization (Array Literal)
int[] scores = {85, 90, 77, 92, 80};
String[] days = {"Monday", "Tuesday", "Wednesday"};

// Accessing elements (zero-indexed)
System.out.println("First score: " + scores[0]); // Output: First score: 85
System.out.println("Second day: " + days[1]);  // Output: Second day: Tuesday

// Modifying elements
scores[0] = 88;
System.out.println("New first score: " + scores[0]); // Output: New first score: 88

// Array length
System.out.println("Number of scores: " + scores.length); // Output: Number of scores: 5

Looping Through Arrays

int[] data = {2, 4, 6, 8, 10};

// Using traditional for loop
System.out.print("Data (traditional for): ");
for (int i = 0; i < data.length; i++) {
    System.out.print(data[i] + " ");
}
System.out.println();

// Using enhanced for loop (for-each)
System.out.print("Data (enhanced for): ");
for (int element : data) {
    System.out.print(element + " ");
}
System.out.println();

Multi-dimensional Arrays

Arrays of arrays.

int[][] matrix = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

System.out.println("Element at row 1, col 2: " + matrix[1][2]); // Output: 6 (0-indexed)

// Looping through a 2D array
for (int i = 0; i < matrix.length; i++) {         // Iterate through rows
    for (int j = 0; j < matrix[i].length; j++) {  // Iterate through columns in current row
        System.out.print(matrix[i][j] + "\t");
    }
    System.out.println();
}

Object-Oriented Programming (OOP) in Java

Java is fundamentally an object-oriented language. OOP concepts help in building modular, reusable, and maintainable software.

1. Classes and Objects

  • Class: A blueprint or template for creating objects. It defines attributes (data/fields) and methods (behavior).
  • Object: An instance of a class. It has its own state (values of attributes) and behavior (defined by methods).
// Define a class 'Dog'
class Dog {
    // Attributes (instance variables / fields)
    String name;
    String breed;
    int age;

    // Constructor: Special method to initialize objects
    public Dog(String name, String breed, int age) {
        this.name = name; // 'this' refers to the current object
        this.breed = breed;
        this.age = age;
        System.out.println(this.name + " the " + this.breed + " is created!");
    }

    // Method (behavior)
    public void bark() {
        System.out.println(name + " says: Woof woof!");
    }

    public void displayDetails() {
        System.out.println("Name: " + name + ", Breed: " + breed + ", Age: " + age);
    }
}

// Main class to run the example
public class DogDemo {
    public static void main(String[] args) {
        // Create Dog objects (instances of the Dog class)
        Dog dog1 = new Dog("Buddy", "Golden Retriever", 3); // 'new' keyword creates an object
        Dog dog2 = new Dog("Lucy", "Poodle", 5);

        // Access attributes (generally not recommended directly, use getters/setters)
        // System.out.println(dog1.name + " is " + dog1.age + " years old.");

        // Call methods
        dog1.bark();
        dog2.bark();

        dog1.displayDetails();
        dog2.displayDetails();
    }
}

Save the above code in two files: Dog.java and DogDemo.java, or combine them in one file named DogDemo.java if `Dog` is not public (or is an inner class). For simplicity, if `Dog` is a top-level class, it should be in its own `Dog.java` file if it's public. If combined, only one top-level class can be `public` and must match the filename.

2. Encapsulation

Bundling data (attributes) and methods that operate on the data within a single unit (class). It also involves restricting direct access to some of an object's components (data hiding).

Achieved using access modifiers and getter/setter methods.

  • public: Accessible from any other class.
  • private: Accessible only within the declared class.
  • protected: Accessible within the same package and by subclasses.
  • Default (no modifier): Accessible only within the same package.
class Person {
    private String name; // Private attribute
    private int age;     // Private attribute

    // Constructor
    public Person(String name, int age) {
        this.name = name;
        setAge(age); // Use setter for validation
    }

    // Getter for name
    public String getName() {
        return name;
    }

    // Setter for name
    public void setName(String name) {
        this.name = name;
    }

    // Getter for age
    public int getAge() {
        return age;
    }

    // Setter for age (with validation)
    public void setAge(int age) {
        if (age > 0) {
            this.age = age;
        } else {
            System.out.println("Age must be positive.");
            this.age = 0; // Default or handle error
        }
    }
}

public class EncapsulationDemo {
    public static void main(String[] args) {
        Person person1 = new Person("Alice", 30);
        System.out.println("Name: " + person1.getName()); // Access via getter
        // person1.age = -5; // Error: age has private access

        person1.setAge(-5); // Tries to set invalid age
        System.out.println("Age: " + person1.getAge()); // Will be 0 or show error msg
        person1.setAge(31);
        System.out.println("New Age: " + person1.getAge());
    }
}

3. Inheritance

Allows a new class (subclass or child class) to inherit attributes and methods from an existing class (superclass or parent class). Promotes code reuse.

Uses the extends keyword.

// Superclass
class Animal {
    String name;

    public Animal(String name) {
        this.name = name;
    }

    public void eat() {
        System.out.println(name + " is eating.");
    }

    public void makeSound() {
        System.out.println(name + " makes a sound.");
    }
}

// Subclass inheriting from Animal
class Cat extends Animal {
    public Cat(String name) {
        super(name); // Calls the constructor of the superclass (Animal)
    }

    // Overriding the makeSound method from Animal
    @Override // Good practice annotation
    public void makeSound() {
        System.out.println(name + " says: Meow!");
    }

    public void purr() {
        System.out.println(name + " is purring.");
    }
}

public class InheritanceDemo {
    public static void main(String[] args) {
        Animal myAnimal = new Animal("Generic Animal");
        myAnimal.eat();
        myAnimal.makeSound();

        Cat myCat = new Cat("Whiskers");
        myCat.eat();         // Inherited from Animal
        myCat.makeSound();   // Overridden in Cat
        myCat.purr();        // Specific to Cat
    }
}

super keyword is used to call superclass constructors or methods.

@Override annotation indicates that a method is intended to override a method in a superclass. Helps catch errors if the signature doesn't match.

4. Polymorphism ("Many Forms")

The ability of an object to take on many forms. In Java, it often refers to a superclass reference variable holding a subclass object.

Allows you to perform a single action in different ways.

  • Compile-time Polymorphism (Method Overloading): Already covered in the Methods section.
  • Run-time Polymorphism (Method Overriding): When a subclass provides a specific implementation for a method already defined in its superclass. The actual method executed is determined at runtime based on the object's type.
// Using Animal and Cat classes from Inheritance example

public class PolymorphismDemo {
    public static void main(String[] args) {
        Animal animal1 = new Animal("Creature");
        Animal animal2 = new Cat("Tom"); // Superclass reference, Subclass object

        animal1.makeSound(); // Calls Animal's makeSound
        animal2.makeSound(); // Calls Cat's makeSound (Runtime Polymorphism)

        // animal2.purr(); // Error: purr() is not in Animal class.
        // To call purr(), you need to cast animal2 to Cat type
        if (animal2 instanceof Cat) {
            Cat tomCat = (Cat) animal2; // Downcasting
            tomCat.purr();
        }
    }
}

5. Abstraction (Brief)

Hiding complex implementation details and showing only essential features of the object.

Achieved using abstract classes and interfaces.

  • Abstract Class: Cannot be instantiated. Can have abstract methods (methods without a body) and concrete methods. Subclasses must implement abstract methods. 
    abstract class Shape {
    public abstract double area(); // Abstract method
    public void display() { System.out.println("This is a shape."); }
}

class Circle extends Shape {
    double radius;
    public Circle(double r) { this.radius = r; }
    @Override
    public double area() { return Math.PI * radius * radius; }
}
  • Interface: A completely abstract "class" that can only contain abstract methods (by default, pre-Java 8) and static final constants. A class can implement multiple interfaces. (Java 8+ allows default and static methods in interfaces). 
    interface Drawable {
    void draw(); // Abstract method (public abstract by default)
}

class Rectangle implements Drawable {
    @Override
    public void draw() { System.out.println("Drawing a rectangle."); }
}

Exception Handling

An exception is an event that occurs during the execution of a program that disrupts the normal flow of instructions. Java provides a robust mechanism to handle exceptions using try-catch-finally blocks.

public class ExceptionExample {
    public static void main(String[] args) {
        try {
            // Code that might throw an exception
            int[] numbers = {1, 2, 3};
            System.out.println(numbers[5]); // ArrayIndexOutOfBoundsException
            
            // int result = 10 / 0; // ArithmeticException
        } catch (ArrayIndexOutOfBoundsException e) {
            System.err.println("Error: Array index is out of bounds.");
            // e.printStackTrace(); // Prints stack trace for debugging
        } catch (ArithmeticException e) {
            System.err.println("Error: Cannot divide by zero.");
        } catch (Exception e) { // Generic catch block (catches any other exception)
            System.err.println("An unexpected error occurred: " + e.getMessage());
        } finally {
            // This block is always executed, whether an exception occurred or not.
            // Useful for cleanup (e.g., closing files, database connections).
            System.out.println("The 'finally' block is executed.");
        }
        System.out.println("Program continues after exception handling.");
    }
}

Keywords:

  • try: The block of code to be monitored for exceptions.
  • catch: Catches and handles a specific type of exception. Multiple catch blocks can be used.
  • finally: Always executed. Optional.
  • throw: Used to manually throw an exception.
  • throws: Used in a method signature to declare exceptions that the method might throw (and doesn't handle itself).

Brief Introduction to Java Collections Framework

The Java Collections Framework (JCF) provides a set of interfaces and classes to represent and manipulate collections of objects, like lists, sets, and maps. It offers more flexibility and functionality than basic arrays.

This is a large topic. Here's a very brief overview of common interfaces:

  • List: An ordered collection (sequence) that allows duplicate elements.
    • Common implementation: ArrayList (dynamic array), LinkedList.
    import java.util.ArrayList;
import java.util.List;

public class ListExample {
    public static void main(String[] args) {
        List<String> names = new ArrayList<>(); // Using Generics for type safety
        names.add("Alice");
        names.add("Bob");
        names.add("Charlie");
        names.add("Alice"); // Duplicates allowed

        System.out.println(names); // Output: [Alice, Bob, Charlie, Alice]
        System.out.println("Element at index 1: " + names.get(1)); // Bob

        for (String name : names) {
            System.out.println(name);
        }
    }
}
  • Set: A collection that does not allow duplicate elements.
    • Common implementations: HashSet (unordered), TreeSet (sorted).
    import java.util.HashSet;
import java.util.Set;

public class SetExample {
    public static void main(String[] args) {
        Set<Integer> numbers = new HashSet<>();
        numbers.add(10);
        numbers.add(20);
        numbers.add(10); // Duplicate, will not be added
        numbers.add(30);

        System.out.println(numbers); // Output: e.g., [20, 10, 30] (order not guaranteed for HashSet)
    }
}
  • Map: A collection that stores key-value pairs. Keys must be unique.
    • Common implementations: HashMap (unordered), TreeMap (sorted by key).
    import java.util.HashMap;
import java.util.Map;

public class MapExample {
    public static void main(String[] args) {
        Map<String, Integer> studentScores = new HashMap<>();
        studentScores.put("Alice", 95);
        studentScores.put("Bob", 88);
        studentScores.put("Charlie", 92);
        studentScores.put("Alice", 98); // Updates Alice's score

        System.out.println(studentScores); // Output: e.g., {Bob=88, Alice=98, Charlie=92}
        System.out.println("Bob's score: " + studentScores.get("Bob"));

        for (Map.Entry<String, Integer> entry : studentScores.entrySet()) {
            System.out.println(entry.getKey() + ": " + entry.getValue());
        }
    }
}

Using Generics (e.g., List<String>) provides type safety at compile time.

Brief Introduction to File I/O

Java provides classes in the java.io package for file input and output operations.

Always handle IOException when working with files.

Writing to a File

import java.io.FileWriter;
import java.io.BufferedWriter;
import java.io.IOException;

public class WriteToFile {
    public static void main(String[] args) {
        String fileName = "output.txt";
        // Try-with-resources statement ensures the writer is closed automatically
        try (BufferedWriter writer = new BufferedWriter(new FileWriter(fileName))) {
            writer.write("Hello from Java File I/O!");
            writer.newLine(); // Writes a platform-dependent line separator
            writer.write("This is the second line.");
            System.out.println("Successfully wrote to the file.");
        } catch (IOException e) {
            System.err.println("An error occurred while writing to the file: " + e.getMessage());
            // e.printStackTrace();
        }
    }
}

Reading from a File

import java.io.FileReader;
import java.io.BufferedReader;
import java.io.IOException;
import java.io.FileNotFoundException;

public class ReadFromFile {
    public static void main(String[] args) {
        String fileName = "output.txt"; // Assuming this file was created by WriteToFile
        // Try-with-resources
        try (BufferedReader reader = new BufferedReader(new FileReader(fileName))) {
            String line;
            System.out.println("Content of the file:");
            while ((line = reader.readLine()) != null) { // Read line by line
                System.out.println(line);
            }
        } catch (FileNotFoundException e) {
            System.err.println("Error: File not found - " + fileName);
        } catch (IOException e) {
            System.err.println("An error occurred while reading the file: " + e.getMessage());
        }
    }
}

The try-with-resources statement (introduced in Java 7) is highly recommended as it automatically closes resources (like file streams) when the block is exited, even if exceptions occur.
