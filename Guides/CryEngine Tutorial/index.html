CryEngine Beginner's Guide: Crafting Your First World

Introduction to CryEngine

Welcome to the CryEngine Beginner's Guide! CryEngine is a powerful, real-time game development platform renowned for its cutting-edge visuals, realistic physics, and extensive toolset, particularly excelling in creating large, open-world environments.

This guide will introduce you to the fundamental aspects of CryEngine, helping you take your first steps from installation to crafting a basic interactive environment.

Note: CryEngine is a feature-rich engine that evolves. While this guide focuses on core principles applicable to CryEngine 5.x, specific UI elements or workflows might see minor changes in newer updates.

1. Installation & Setup

CryEngine is managed and installed via the CRYENGINE Launcher.

  1. Download CRYENGINE Launcher: Visit the official CryEngine website and download the CRYENGINE Launcher.
  2. Install Launcher & Sign Up: Run the installer. You will need to create a free CryEngine account to log in and use the launcher.
  3. Install the Engine:
    • Once logged into the launcher, navigate to the "Engines" or "My Engines" tab.
    • Click on "Install Engine" and choose the latest stable version (e.g., CRYENGINE 5.6, 5.7, or newer).
    • The launcher will download and install the selected engine version. This may take some time.
  4. Create a New Project:
    • Go to the "My Projects" tab in the launcher.
    • Click "New Project".
    • Select the engine version you just installed.
    • Choose a project template. The "Blank (C++)" or "Blank (C# - if available and preferred)" template is a good starting point for learning. The "GameSDK Sample Project" offers a more complete example to explore later.
    • Give your project a name and choose a location, then click "Create".
  5. Launch the Editor: Once the project is created, you can open it by selecting it in "My Projects" and clicking "Launch Editor".

2. CryEngine Editor (Sandbox) Overview

The CryEngine editor, traditionally known as Sandbox, is where you'll build your worlds and game experiences. It's a comprehensive suite of tools. Here are some key windows and areas:

  • Perspective Viewport: The main 3D window where you view and interact with your level. Navigation is typically:
    • Hold Right Mouse Button: Look around.
    • Right Mouse Button + WASD: Fly/Move. Q/E for up/down. Mouse Wheel to adjust speed.
    • Middle Mouse Button: Pan.
  • RollupBar / Object Panel: (Often on the right or as a tabbed window) This is crucial. It contains tabs for:
    • Objects: Create and place various entity types (Brushes, Entities, Lights, Sounds, etc.).
    • Terrain: Tools for sculpting terrain, painting layers, modifying heightmaps.
    • Layers: Manage object layers for organization.
    • Vegetation: Paint and manage vegetation objects.
    • And more, depending on the selected tool mode.
  • Asset Browser: (Usually View > Open View Pane > Asset Browser) For browsing and managing all project assets like models (.cgf), materials (.mtl), textures, animations.
  • Properties Panel / Entity Properties: (Often bottom right or as a tabbed window) Displays and allows editing of parameters for the currently selected object or tool.
  • Console: (Access with `~` tilde key) For commands, debugging information, and error messages.
  • Specialized Editors: CryEngine has many dedicated editors accessible from the main menu (e.g., Tools > Material Editor, Tools > Character Tool, Tools > Flow Graph Editor).
  • Toolbar: Contains common tools like Translate, Rotate, Scale, object selection modes, and shortcuts to various editors.

The editor layout is highly customizable. You can dock, float, and rearrange windows to suit your workflow. Explore View > Layouts for presets.

3. Levels, Layers & Objects

Levels

In CryEngine, a game "scene" or "map" is referred to as a Level. Each level is typically stored as a .cry file within your project's Levels folder.

  • To create a new level: File > New.
  • To open an existing level: File > Open.
  • Save your level frequently: File > Save.

Layers

Layers are essential for organizing the objects within your level. Think of them as folders for your game objects.

  • Access layers via the "Layers" tab in the RollupBar or a dedicated Layers window (View > Open View Pane > Layers).
  • You can create new layers, assign objects to layers, and control the visibility or frozen state of entire layers.
  • Good layer management is crucial for performance and organization in complex levels. For example, create layers for "StaticProps_Buildings", "Vegetation_Trees", "Gameplay_Triggers", etc.

Objects (Entities)

Everything in your level is an object, often referred to as an Entity in CryEngine terminology.

  • Brushes: Basic geometric primitives (cubes, spheres, etc.) often used for level blockouts or simple static geometry. Found under the "Objects" tab in RollupBar > "Brush".
  • Entities: More complex objects that can have scripts, components, and specific behaviors. This includes characters, vehicles, lights, sound emitters, gameplay triggers, and imported 3D models. Found under "Objects" tab > "Entity".
    • When you place an "Entity" of type "GeomEntity", you can assign a 3D model (.cgf file) to it.
  • Decals: Textures projected onto surfaces (e.g., bullet holes, stains, posters).
  • Lights: Various light types to illuminate your scene (Point, Spot, Projector, EnvironmentProbe).

To place an object, select its type from the RollupBar, configure its properties (if any), and click in the Perspective Viewport.

4. Terrain Sculpting & Painting

CryEngine is famous for its powerful terrain tools, allowing for the creation of vast and detailed landscapes.

  1. Create Terrain: If your new level doesn't have terrain, go to Terrain > Edit Terrain. This opens the Terrain Editor panel (often docked with the RollupBar). Click File > New Terrain within this panel to generate a default heightmap.
  2. Sculpting Tools: In the Terrain Editor panel, under the "Modify" section, you'll find tools like:
    • Raise/Lower: To sculpt hills and valleys. Adjust brush size and hardness.
    • Smooth: To soften sharp edges.
    • Flatten: To create flat areas at a specific height.
    • Noise: To add random detail.
  3. Terrain Layers (Texturing):
    • Go to Terrain > Layer Painter (or find the tab in the Terrain Editor).
    • A default terrain material is usually assigned. You can modify this in the Material Editor or assign a new one.
    • In the Layer Painter, you can add new texture layers (e.g., grass, rock, sand, dirt). Each layer uses a material and can have properties like surface type (for footstep sounds, physics interactions).
    • Select a layer and a brush, then paint directly onto the terrain in the viewport. You can control opacity, hardness, and altitude/slope constraints for painting.
  4. Holes: You can create holes in the terrain (for caves, etc.) using the "Hole" tool in the Terrain Editor.

Large, high-resolution terrains can be performance-intensive. Optimize by using appropriate heightmap resolutions and level of detail (LOD) settings.

5. Vegetation & Environment

CryEngine's vegetation system allows for efficient placement of vast amounts of foliage and other environmental details.

  1. Open Vegetation Panel: Go to the "Vegetation" tab in the RollupBar (or Terrain > Vegetation).
  2. Add Vegetation Objects:
    • Click "Add Object".
    • Browse to a vegetation asset (.cgf file, e.g., a tree or a rock model) in the Asset Browser or file dialog. You can find many sample assets in the `GameSDK` project or on the CRYENGINE Marketplace.
    • The selected object will appear in the list in the Vegetation panel.
  3. Paint Vegetation:
    • Select one or more vegetation objects from the list (hold Ctrl to select multiple).
    • Adjust parameters like Size, Randomness, Density, Slope alignment, etc.
    • Select the "Paint" tool (brush icon) in the Vegetation panel.
    • Click and drag in the Perspective Viewport to paint instances of the selected vegetation onto the terrain or other objects.
  4. Other Tools: The Vegetation panel also includes tools to Erase, Move, Rotate, and individually place vegetation instances.

The vegetation system uses instancing and LODs to render large amounts of foliage efficiently. Ensure your vegetation assets are optimized with proper LODs.

Beyond vegetation, you can enhance your environment with:

  • Environment Probes: Capture reflections for PBR materials. (Found under "Objects" tab > "Area" > "EnvironmentProbe").
  • Decals: Add details like cracks, stains, or posters.
  • Particles: Create effects like fire, smoke, waterfalls (using the Particle Editor: Tools > Particle Editor).

6. Flow Graph (Visual Scripting)

Flow Graph is CryEngine's visual scripting system. It allows you to create game logic and interactions without writing traditional code by connecting nodes that represent functions and events.

  1. Open Flow Graph Editor: Tools > Flow Graph Editor or press F.
  2. Create a Flow Graph:
    • Select an entity in your level that will host the Flow Graph (e.g., a trigger volume, a specific interactive object). You can also create global Flow Graphs.
    • In the Flow Graph Editor, right-click in the "Graphs" pane (left side) and select "Add FlowGraph" or "Add Entity FlowGraph" if an entity is selected. Give it a name.
  3. Adding Nodes:
    • Right-click in the main graph view (center area) to open the context menu.
    • Browse through categories like "Entity", "Game", "Input", "Math", "Time" to find nodes. For example, to react to a player entering a trigger: Entity > ProximityTrigger. To start an action: Game > Start.
  4. Connecting Nodes: Nodes have input ports (usually on the left) and output ports (usually on the right). Click and drag from an output port of one node to an input port of another to create a connection (a "link"). This defines the flow of logic and data.
  5. Example: Door Opens on Player Proximity
    1. Place a door entity and a ProximityTrigger entity in your level.
    2. Select the ProximityTrigger, create an Entity FlowGraph for it.
    3. Add an Entity:ProximityTrigger node. It will have outputs like "Enter" and "Leave".
    4. Add an Entity:PlayAnimation node (or similar action for your door, like Entity:SetPos or a custom door component node).
    5. In the Entity:PlayAnimation node's properties (select node, view in Properties panel), assign the Door entity to its "EntityId" input or by picking it. Set the animation name (e.g., "door_open").
    6. Connect the "Enter" output of the ProximityTrigger node to the "Start" (or "Play") input of the PlayAnimation node.

Flow Graph is powerful for prototyping, level scripting, and simple AI behaviors. For more complex systems, C++ or Schematyc is generally preferred.

7. C++ & Schematyc Scripting

For advanced game logic, performance-critical systems, and custom engine extensions, C++ is the primary programming language in CryEngine.

C++

Direct C++ scripting involves creating new components, game objects, and systems by deriving from CryEngine's base classes.

  • You'll need a C++ IDE like Visual Studio (Community Edition is free).
  • When you create a C++ project, CryEngine generates a Visual Studio solution.
  • Key concepts include:
    • Entities and Components: Modern CryEngine uses an Entity-Component System (ECS) architecture. You create custom components in C++ and attach them to entities.
    • Game Object Extensions: For older systems, or specific gameplay objects not fitting the component model.
    • CryCommon Headers: Include files providing access to engine APIs.
    • Building Code: Use Visual Studio to compile your C++ code into DLLs that the engine loads.

C++ development in CryEngine requires a good understanding of C++ and game engine architecture. Refer to the official CryEngine C++ API documentation and tutorials.

Schematyc

Schematyc is a node-based visual logic system designed to be more accessible than raw C++ for certain tasks, while still offering robust capabilities. It can bridge the gap between Flow Graph and C++.

  • Schematyc allows you to define custom entity components, state machines, and data structures visually.
  • These visual graphs can then be compiled into efficient C++ code or interpreted directly.
  • It aims to provide a more designer-friendly way to implement complex behaviors that might be cumbersome in Flow Graph but don't require deep C++ knowledge.

Explore the Tools > Schematyc Editor to see how it works. Its usage and prominence can vary between CryEngine versions and project types.

Scripting in CryEngine has a steeper learning curve than in some other engines. Start with Flow Graph for basic interactions and gradually explore Schematyc or C++ as your needs and skills grow.

8. Lighting & Time of Day

CryEngine's lighting system is one of its hallmarks, enabling highly realistic and dynamic illumination.

Basic Lights

You can place various light entities from the "Objects" tab in the RollupBar > "Light":

  • Light (Point Light): Emits light in all directions from a point.
  • Projector (Spot Light): Emits light in a cone shape. Can project textures.
  • EnvironmentProbe: Captures surrounding environment for reflections. Essential for PBR materials.
  • DeferredLight: Optimized lights that work well with deferred rendering.

Each light has numerous properties in the Properties Panel (color, intensity, radius, shadow casting options, etc.).

Time of Day (TOD) System

CryEngine's Time of Day system is incredibly powerful for creating dynamic skies, weather, and lighting transitions.

  1. Open TOD Editor: Terrain > Time Of Day or Tools > Time Of Day Editor.
  2. The editor displays a timeline (usually 00:00 to 24:00). You can scrub through time to see immediate changes in the viewport.
  3. Keyframing Parameters: Almost every atmospheric and lighting parameter can be keyframed along this timeline. This includes:
    • Sun direction, color, intensity.
    • Sky color, fog density and color.
    • Cloud settings.
    • Moon properties.
    • Global illumination parameters.
  4. To set a keyframe: Move the timeline slider to the desired time, adjust a parameter (e.g., Sun Color), and right-click on the parameter's track in the TOD editor to add a key.
  5. You can create multiple TOD sequences (e.g., for different weather states or moods) and switch between them.

Experimenting with the Time of Day editor is crucial for achieving stunning visuals. Save your TOD settings frequently (File > Save within the TOD editor).

9. Packaging Your Game

When your project is ready for distribution, you need to package it into a standalone build.

  1. Project Configurator: Open the CRYENGINE Launcher. Go to your project and click "Configure Project" or a similar option if available. This tool helps manage project settings, assets, and build configurations. (The exact tools and steps can vary slightly between engine versions).
  2. Build Options / Game Exporter:
    • Within the CryEngine Editor, look for options under File > Export to Engine or similar, or through dedicated build tools.
    • Alternatively, the CRYENGINE Launcher or specific command-line tools provided with the engine are used for packaging.
    • You typically need to specify:
      • Target platform (e.g., Windows 64-bit).
      • Build configuration (e.g., "Release" for a final build, "Debug" for testing).
      • Which levels to include.
  3. Asset Processing: Before packaging, ensure all your assets are processed by the Resource Compiler (RC). This usually happens automatically when you save assets or can be triggered manually. The RC converts source assets (like .max files or .tif textures) into engine-ready formats (.cgf, .dds).
  4. Running the Packaged Game: The build process will generate an executable (e.g., YourGame.exe) and all necessary data files in a specified output folder. You can then run this executable to play your game.

Packaging can be a complex step. Consult the official CryEngine documentation for the specific version you are using for detailed instructions on building and exporting your project. Ensure your license agreement with Crytek is understood regarding distribution.
