A Comprehensive Beginner's Guide to HTML

Introduction to HTML

HTML stands for HyperText Markup Language. It is the standard markup language used to create and design web pages and web applications. HTML describes the structure of a web page semantically and originally included cues for the appearance of the document.

Key characteristics of HTML:

  • Markup Language: It uses "tags" to mark up or annotate text, images, and other content for display in a web browser.
  • Structure of Web Pages: HTML forms the backbone or skeleton of a web page, defining elements like headings, paragraphs, links, images, etc.
  • Not a Programming Language: HTML is a markup language, not a programming language. It doesn't have logic like conditional statements or loops. For dynamic content and interactivity, it's used with CSS and JavaScript.
  • Interpreted by Browsers: Web browsers (like Chrome, Firefox, Safari) read HTML documents and render them into visible or audible web pages.

Learning HTML is the first essential step for anyone aspiring to be a web developer or to understand how websites are built.

Setting Up Your Environment

The good news is you don't need much to start writing HTML!

1. Text Editor

You can write HTML in any plain text editor. However, using a code editor will provide features like syntax highlighting, code completion, and better file management, making your life easier.

Popular choices:
  • Visual Studio Code (VS Code): Free, powerful, and highly popular. Download VS Code
  • Sublime Text: Lightweight and fast. Download Sublime Text
  • Atom: Free and open-source, developed by GitHub. Download Atom
  • Notepad++ (Windows): A free, lightweight code editor for Windows. Download Notepad++
  • Basic text editors like Notepad (Windows) or TextEdit (Mac - ensure it's in plain text mode) can also be used.

2. Web Browser

You'll need a web browser to view your HTML pages. Any modern browser will work:

  • Google Chrome
  • Mozilla Firefox
  • Microsoft Edge
  • Safari (macOS)

That's it! No compilers or complex installations are required for HTML itself.

Basic HTML Document Structure

Every HTML document follows a basic structure. Here's a minimal example:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
</head>
<body>
    <!-- Content visible to users goes here -->
    <h1>Hello, World!</h1>
    <p>This is my first paragraph.</p>
</body>
</html>

Explanation of the structure:

  • <!DOCTYPE html>: This declaration defines that the document is an HTML5 document. It must be the very first thing in your HTML document, before any HTML tags. It is not an HTML tag itself but an instruction to the web browser about what version of HTML the page is written in.
  • <html>: This is the root element of an HTML page. The lang="en" attribute specifies the language of the document (English in this case), which is good for accessibility and search engines.
  • <head>: This element contains meta-information about the HTML document, which is not displayed on the page itself. This includes:
    • <meta charset="UTF-8">: Specifies the character encoding for the document. UTF-8 is recommended as it covers almost all characters and symbols.
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Configures the viewport for responsive web design, ensuring the page looks good on all devices.
    • <title>My First Web Page</title>: Sets the title of the HTML document, which is displayed in the browser's title bar or tab.
    • It can also contain links to CSS files, JavaScript files, and other metadata.
  • <body>: This element contains all the visible content of the HTML document, such as headings, paragraphs, images, hyperlinks, tables, lists, etc.

To create your first HTML page, copy the code above, save it in a file named index.html (or any name ending with .html or .htm), and then open that file in your web browser.

HTML Elements and Tags

An HTML document is made up of HTML elements.

HTML elements are typically represented by tags. Tags are keywords surrounded by angle brackets < >.

  • Most HTML elements have an opening tag (e.g., <p>) and a closing tag (e.g., </p>).
  • The content goes between the opening and closing tags.
  • The element consists of the opening tag, the content, and the closing tag. For example: <p>This is a paragraph.</p> is an HTML element.

Void Elements (Empty Elements)

Some HTML elements do not have content or a closing tag. These are called void elements or empty elements. They are typically self-closing or written with just the opening tag.

Examples:
  • <br> (line break)
  • <hr> (horizontal rule)
  • <img src="image.jpg" alt="My Image"> (image)
  • <input type="text"> (input field)

In XHTML (an older, stricter version of HTML), these were written with a trailing slash (e.g., <br />). In HTML5, the trailing slash is optional but often still used for clarity.

Nesting Elements

HTML elements can be nested inside other elements. This means an element can contain other elements as its content.

<body>
    <h1>This is a <em>Main</em> Heading</h1> <!-- <em> is nested inside <h1> -->
    <p>This paragraph contains <strong>important</strong> text.</p>
</body>

Nesting must be done correctly: an inner element must be completely closed before its outer element is closed. Incorrect: <p>This is <strong>bold and <em>italic</strong>.</em></p>
Correct: <p>This is <strong>bold and <em>italic</em></strong>.</p>

HTML Attributes

HTML elements can have attributes. Attributes provide additional information about an element and are always specified in the opening tag.

Attributes usually come in name/value pairs like: name="value".

Examples:
  • The <a> (anchor) tag, used for links, has an href attribute that specifies the URL: 
    <a href="https://www.example.com">Visit Example.com</a>
  • The <img> tag has src (source) and alt (alternative text) attributes: 
    <img src="image.png" alt="A descriptive text for the image">
  • The id attribute provides a unique identifier for an element: 
    <p id="intro-paragraph">This is an introduction.</p>
  • The class attribute specifies one or more class names for an element (used for styling with CSS or selecting with JavaScript): 
    <p class="highlight important">This paragraph is highlighted and important.</p>
  • The style attribute is used for inline CSS styling (though external stylesheets are generally preferred): 
    <p style="color: blue; font-size: 16px;">This text is blue and 16px.</p>

Attribute values should generally be enclosed in double quotes (""). Single quotes ('') are also allowed. If the value itself contains double quotes, you can use single quotes, or vice-versa.

Headings and Paragraphs

Headings

HTML provides six levels of headings, from <h1> to <h6>. <h1> defines the most important heading, and <h6> defines the least important.

<h1>This is Heading 1</h1>
<h2>This is Heading 2</h2>
<h3>This is Heading 3</h3>
<h4>This is Heading 4</h4>
<h5>This is Heading 5</h5>
<h6>This is Heading 6</h6>

Browsers automatically add some white space (a margin) before and after a heading. Headings are block-level elements, meaning they start on a new line and take up the full width available.

Use headings for their semantic meaning (to structure your content logically) rather than just to make text big or bold. Typically, a page should have only one <h1> element for the main title of the page.

Paragraphs

The <p> element defines a paragraph.

<p>This is the first paragraph of text. Browsers automatically add some white space (a margin) before and after a paragraph.</p>
<p>This is another paragraph. Paragraphs are also block-level elements.</p>

Browsers will automatically remove extra spaces and extra lines in your HTML code. Any number of spaces or newlines in the HTML source will be rendered as a single space in the browser.

Text Formatting Elements

HTML has several elements for defining text with a special meaning or appearance.

  • <strong>: Defines text with strong importance. Browsers typically render this as bold. 
    <p>This text is <strong>important</strong>.</p>
  • <em>: Defines emphasized text. Browsers typically render this as italic. 
    <p>This text is <em>emphasized</em>.</p>
  • <b>: Defines bold text without any extra importance (presentational). 
    <p>This text is <b>bold</b>.</p>
  • <i>: Defines italic text, often used for technical terms, thoughts, etc. (presentational). 
    <p>This text is in <i>italics</i>.</p>
  • <u>: Defines underlined text. Use with caution as users might mistake it for a hyperlink. 
    <p>This text is <u>underlined</u>.</p>
  • <mark>: Defines marked or highlighted text. 
    <p>Do not forget to buy <mark>milk</mark> today.</p>
  • <small>: Defines smaller text. 
    <p>This is some <small>smaller text</small>.</p>
  • <del>: Defines deleted text (often rendered with a strikethrough). 
    <p>My favorite color is <del>blue</del> red.</p>
  • <ins>: Defines inserted text (often rendered as underlined). 
    <p>My favorite color is <del>blue</del> <ins>red</ins>.</p>
  • <sub>: Defines subscripted text. 
    <p>H<sub>2</sub>O is water.</p>
  • <sup>: Defines superscripted text. 
    <p>E = mc<sup>2</sup>.</p>
  • <br>: Inserts a single line break (a void element). 
    <p>This is some text.<br>This text is on a new line.</p>
  • <hr>: Defines a thematic break or horizontal rule (a void element). Often used to separate content. 
    <p>First section of content.</p>
<hr>
<p>Second section of content.</p>

It's generally recommended to use semantic elements like <strong> and <em> over purely presentational ones like <b> and <i> when the meaning implies importance or emphasis. Styling should primarily be handled by CSS.

Links (Hyperlinks)

The HTML <a> (anchor) tag defines a hyperlink, which is used to link from one page to another, or to a specific part of a page.

The most important attribute of the <a> element is the href attribute, which indicates the link's destination.

Absolute URLs vs. Relative URLs

  • Absolute URL: Links to an external website (e.g., https://www.google.com). 
    <a href="https://www.wikipedia.org">Visit Wikipedia</a>
  • Relative URL: Links to a file within your own website.
    • Link to a page in the same directory: 
      <a href="about.html">About Us</a>
    • Link to a page in a subdirectory: 
      <a href="products/product1.html">View Product 1</a>
    • Link to a page in a parent directory: 
      <a href="../services.html">Our Services</a>

The target Attribute

The target attribute specifies where to open the linked document.

  • _self: Default. Opens the document in the same window/tab as it was clicked.
  • _blank: Opens the document in a new window or tab. 
    <a href="https://www.example.com" target="_blank">Open Example in New Tab</a>
  • _parent: Opens the document in the parent frame.
  • _top: Opens the document in the full body of the window.

When using target="_blank", it's good practice to also add rel="noopener noreferrer" for security and privacy reasons to prevent the new page from accessing your page's window.opener object.

<a href="https://www.example.com" target="_blank" rel="noopener noreferrer">Secure Link in New Tab</a>

Linking to an Email Address

Use mailto: within the href attribute.

<a href="mailto:someone@example.com">Send Email</a>

Linking to a Specific Part of a Page (Bookmarks/Anchors)

First, create an id attribute on the element you want to link to:

<h2 id="section2">Section 2</h2>
<p>Content of section 2...</p>

Then, create a link to that id using a hash (#) symbol:

<a href="#section2">Go to Section 2</a>

You can also link to an ID on another page: <a href="anotherpage.html#section2">...</a>

Images

The HTML <img> tag is used to embed an image in an HTML page. It is a void element (it has no closing tag).

Key attributes for <img>:

  • src: (Source) Specifies the path to the image. This is mandatory.
  • alt: (Alternative text) Provides a textual description of the image. This is crucial for accessibility (e.g., for screen readers) and will be displayed if the image cannot be loaded. This is also mandatory for valid HTML.
<img src="images/my_puppy.jpg" alt="A cute golden retriever puppy playing in the grass">

Image Path

Similar to links, image paths can be absolute or relative:

  • Absolute URL: <img src="https://www.example.com/images/logo.png" alt="Example Logo">
  • Relative URL (image in a subdirectory named "images"): <img src="images/photo.jpg" alt="A photo">
  • Relative URL (image in the same directory): <img src="icon.gif" alt="An icon">

Width and Height Attributes

You can use the width and height attributes to specify the size of the image in pixels.

<img src="landscape.jpg" alt="Beautiful landscape" width="500" height="300">

While you can set width and height using HTML attributes, it's generally better practice to style image dimensions using CSS for more flexibility and maintainability, especially for responsive design. However, specifying width and height in HTML can help the browser reserve space for the image while it loads, preventing page layout shifts.

Image Formats

Common web image formats include:
  • JPEG (.jpg, .jpeg): Good for photographs, supports lossy compression.
  • PNG (.png): Good for graphics with transparency or sharp lines, supports lossless compression.
  • GIF (.gif): Good for simple animations and graphics with limited colors.
  • SVG (.svg): Scalable Vector Graphics, resolution-independent, good for logos and icons.
  • WebP (.webp): A modern format with excellent compression and quality, supports both lossy/lossless and transparency/animation. Browser support is now widespread.

Lists

HTML offers several ways to create lists of items.

Unordered Lists

An unordered list starts with the <ul> tag. Each list item starts with the <li> tag. List items are typically marked with bullets (small black circles by default).

<h3>My Favorite Fruits</h3>
<ul>
    <li>Apple</li>
    <li>Banana</li>
    <li>Orange</li>
</ul>

Ordered Lists

An ordered list starts with the <ol> tag. Each list item starts with the <li> tag. List items are typically marked with numbers by default.

<h3>Steps to Make Tea</h3>
<ol>
    <li>Boil water.</li>
    <li>Put tea bag in cup.</li>
    <li>Pour boiling water into cup.</li>
    <li>Steep for 3-5 minutes.</li>
    <li>Remove tea bag and enjoy.</li>
</ol>

The type attribute can be used on <ol> to change the numbering style (e.g., type="A" for uppercase letters, type="i" for lowercase Roman numerals). However, styling list markers is often better done with CSS.

Description Lists (Definition Lists)

A description list is a list of terms, with a description of each term. It starts with the <dl> tag.

  • <dt>: Defines the term (name).
  • <dd>: Describes each term.
<h3>Web Technologies</h3>
<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language, used for structuring web content.</dd>
    <dt>CSS</dt>
    <dd>Cascading Style Sheets, used for styling web content.</dd>
    <dt>JavaScript</dt>
    <dd>A programming language used for adding interactivity to web pages.</dd>
</dl>

Nesting Lists

You can nest lists within other lists.

<ul>
    <li>Fruit
        <ul>
            <li>Apple</li>
            <li>Banana</li>
        </ul>
    </li>
    <li>Vegetable
        <ol>
            <li>Carrot</li>
            <li>Broccoli</li>
        </ol>
    </li>
</ul>

Tables

HTML tables are used to display data in a tabular format (rows and columns).

Key table elements:

  • <table>: Defines the table.
  • <tr>: Defines a table row.
  • <th>: Defines a table header cell. Text within <th> is typically bold and centered by default.
  • <td>: Defines a table data cell.
<table border="1"> <!-- 'border' attribute is for basic styling, CSS is preferred -->
    <tr>
        <th>First Name</th>
        <th>Last Name</th>
        <th>Age</th>
    </tr>
    <tr>
        <td>Jill</td>
        <td>Smith</td>
        <td>50</td>
    </tr>
    <tr>
        <td>Eve</td>
        <td>Jackson</td>
        <td>94</td>
    </tr>
</table>

Historically, tables were sometimes used for page layout. This is now considered bad practice. Use CSS for layout. Tables should be used for displaying tabular data.

More Table Elements

  • <caption>: Defines a table caption. 
    <table border="1">
    <caption>Monthly Savings</caption>
    <!-- ... rest of table ... -->
</table>
  • <thead>: Groups header content in a table.
  • <tbody>: Groups body content in a table.
  • <tfoot>: Groups footer content in a table.
<table border="1">
    <thead>
        <tr>
            <th>Item</th>
            <th>Price</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Apples</td>
            <td>$2.00</td>
        </tr>
        <tr>
            <td>Bananas</td>
            <td>$1.50</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td>Total</td>
            <td>$3.50</td>
        </tr>
    </tfoot>
</table>

Spanning Columns and Rows

  • colspan attribute (on <th> or <td>): Makes a cell span multiple columns.
  • rowspan attribute (on <th> or <td>): Makes a cell span multiple rows.
<table border="1">
    <tr>
        <th>Name</th>
        <th colspan="2">Contact</th>
    </tr>
    <tr>
        <td>John Doe</td>
        <td>555-1234</td>
        <td>john@example.com</td>
    </tr>
    <tr>
        <th rowspan="2">Address</th>
        <td colspan="2">123 Main St</td>
    </tr>
    <tr>
        <td colspan="2">Anytown, USA</td>
    </tr>
</table>

Forms (Introduction)

HTML forms are used to collect user input. Forms are essential for user interaction, such as login pages, contact forms, search boxes, etc.

The <form> element is a container for different types of input elements.

Key Form Elements

  • <form>: The container for the form.
    • action attribute: Specifies where to send the form-data when a form is submitted (usually a server-side script URL).
    • method attribute: Specifies the HTTP method to use when submitting the form-data (commonly GET or POST).
  • <label>: Defines a label for an <input> element. Improves accessibility. The for attribute of the label should match the id attribute of the input element.
  • <input>: The most versatile form element. The type attribute determines how it looks and behaves. Common types:
    • text: Single-line text input.
    • password: Password field (characters are masked).
    • email: For email addresses (provides some client-side validation).
    • number: For numerical input.
    • date: For selecting a date.
    • checkbox: Allows users to select zero or more options from a set.
    • radio: Allows users to select only one option from a set. Radio buttons with the same name attribute are grouped.
    • submit: A button for submitting the form.
    • reset: A button to reset form fields to their initial values.
    • file: For uploading files.
    Other common attributes: name (essential for server-side processing), id (for labels and JavaScript), value, placeholder, required.
  • <textarea>: Defines a multi-line text input area. 
    <textarea name="message" rows="4" cols="50">Enter your message here...</textarea>
  • <select> (Dropdown List): Creates a dropdown list.
    • <option>: Defines an option within the <select> list. The value attribute of the option is what gets sent to the server.
    <label for="cars">Choose a car:</label>
<select id="cars" name="cars">
  <option value="volvo">Volvo</option>
  <option value="saab">Saab</option>
  <option value="mercedes" selected>Mercedes</option> <!-- 'selected' makes it default -->
  <option value="audi">Audi</option>
</select>
  • <button>: Defines a clickable button. Can be of type="submit", type="reset", or type="button" (generic button, often used with JavaScript). 
    <button type="submit">Send</button>

Basic Form Example

<form action="/submit-form.php" method="post">
    <div>
        <label for="username">Username:</label>
        <input type="text" id="username" name="username" required>
    </div>
    <div>
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" placeholder="you@example.com">
    </div>
    <div>
        <label>Gender:</label>
        <input type="radio" id="male" name="gender" value="male">
        <label for="male">Male</label>
        <input type="radio" id="female" name="gender" value="female">
        <label for="female">Female</label>
    </div>
    <div>
        <input type="checkbox" id="subscribe" name="subscribe" value="yes" checked>
        <label for="subscribe">Subscribe to newsletter</label>
    </div>
    <div>
        <label for="comments">Comments:</label>
        <textarea id="comments" name="comments" rows="3"></textarea>
    </div>
    <button type="submit">Submit Application</button>
    <button type="reset">Clear Form</button>
</form>

Form processing (what happens when you click "Submit") typically requires a server-side language (like PHP, Python, Node.js, Java, Ruby, etc.) or can be handled client-side with JavaScript for some simpler tasks or AJAX submissions.

Semantic HTML

Semantic HTML elements are elements with a meaning. They clearly describe their meaning to both the browser and the developer. Using semantic HTML improves accessibility, SEO, and code readability.

Examples of non-semantic elements: <div> and <span>. They tell nothing about their content.

Examples of semantic elements (many introduced in HTML5):

  • <header>: Defines a header for a document or a section. Often contains a logo, navigation, search bar.
  • <nav>: Defines a set of navigation links.
  • <main>: Specifies the main content of a document. There should be only one <main> element per page.
  • <article>: Defines self-contained content that could be distributed independently (e.g., a blog post, news story, forum post).
  • <section>: Defines a thematic grouping of content, typically with a heading.
  • <aside>: Defines content aside from the content it is placed in (e.g., a sidebar, call-out box).
  • <footer>: Defines a footer for a document or a section. Often contains copyright information, contact details, sitemap links.
  • <figure> and <figcaption>: Used to group an image, diagram, or code listing with a caption. 
    <figure>
  <img src="chart.png" alt="Sales chart for Q1">
  <figcaption>Fig.1 - Quarterly Sales Chart.</figcaption>
</figure>
  • <time>: Defines a specific time (or date). The datetime attribute provides a machine-readable format. 
    <p>Published on <time datetime="2023-10-26">October 26, 2023</time>.</p>

Example Page Layout with Semantic HTML

<!DOCTYPE html>
<html lang="en">
<head>
    <title>Semantic Page Layout</title>
</head>
<body>
    <header>
        <h1>My Awesome Website</h1>
        <nav>
            <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">About</a></li>
                <li><a href="#">Contact</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <article>
            <h2>Main Article Title</h2>
            <p>This is the main content of the article...</p>
        </article>

        <section>
            <h2>Related Posts</h2>
            <p>Some related information here...</p>
        </section>
    </main>

    <aside>
        <h3>Sidebar Content</h3>
        <p>Advertisements or related links.</p>
    </aside>

    <footer>
        <p>&copy; 2023 My Awesome Website. All rights reserved.</p>
    </footer>
</body>
</html>

HTML Comments

Comments are used to add notes or explanations within your HTML code. They are ignored by the browser and not displayed on the page.

HTML comments start with <!-- and end with -->.

<!-- This is a single-line comment -->

<p>This is a paragraph.</p>

<!--
  This is a
  multi-line comment.
  It can span several lines.
-->

<!-- You can also use comments to temporarily hide parts of your code:
<p>This paragraph is commented out and will not be displayed.</p>
-->

Comments are useful for:

  • Explaining complex code sections.
  • Leaving notes for yourself or other developers.
  • Temporarily disabling code for testing or debugging.

HTML Entities

Some characters are reserved in HTML (e.g., <, >, &). To display these characters literally in your text content, you need to use HTML entities.

An HTML entity starts with an ampersand (&) and ends with a semicolon (;).

Common HTML Entities:

  • &lt; for < (less than)
  • &gt; for > (greater than)
  • &amp; for & (ampersand)
  • &quot; for " (double quote)
  • &apos; for ' (single quote - not always necessary, but good for consistency)
  • &nbsp; for a non-breaking space (prevents a line break at that space)
  • &copy; for © (copyright symbol)
  • &reg; for ® (registered trademark symbol)
  • &trade; for ™ (trademark symbol)
<p>To write <p> in text, use &lt;p&gt;.</p>
<p>This text contains a non-breaking&nbsp;space.</p>
<p>Copyright &copy; 2023.</p>

Entities can also be represented by their numeric character reference (e.g., &#60; for <).

Next Steps

Congratulations on completing this introductory guide to HTML! You now have a solid understanding of how to structure web pages.

To continue your web development journey:

  • Practice: The more you write HTML, the more comfortable you'll become. Try rebuilding simple web page structures.
  • Learn CSS (Cascading Style Sheets): HTML defines the structure, but CSS defines the presentation (layout, colors, fonts, etc.). They work hand-in-hand. This is the most logical next step.
  • Learn JavaScript: Once you have structure (HTML) and style (CSS), JavaScript adds interactivity and dynamic behavior to your web pages.
  • Validate Your HTML: Use tools like the W3C Markup Validation Service to check your HTML code for errors and ensure it conforms to standards.
  • Explore More HTML Elements and Attributes: There are many more elements for embedding multimedia (<audio>, <video>, <iframe>), advanced form controls, and more semantic tags. Refer to resources like MDN Web Docs.
  • Understand Web Accessibility (a11y): Learn how to write HTML that is accessible to all users, including those with disabilities. This includes using semantic HTML correctly, providing alt text for images, and ensuring keyboard navigability.
  • Build Simple Websites: Start combining your HTML, CSS, and eventually JavaScript skills to build small, complete websites.
  • Refer to MDN Web Docs: The Mozilla Developer Network (MDN) is an excellent and comprehensive resource for all web technologies, including HTML.

HTML is the foundation of the web. A strong grasp of HTML will serve you well in any web-related field.

Happy coding!
